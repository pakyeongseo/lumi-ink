# 개발 빌드 v66.30 · 정규식 미리보기·테마·자유 메모 삽입

> 사용자 배포 표기는 **v1.5**를 유지합니다.
> 개발 빌드: **v66.30**

## 정규식 작업실

- 세 번째 작성 단계의 라벨을 `Sample`로 바로잡았습니다.
- EDIT / PREVIEW / SPLIT 탭 아래에 넓게 깔리던 어두운 그라데이션 바탕을 제거하고, 문서 바탕·옅은 경계선만 남겼습니다.
- 옵션 제목을 더 작은 글자와 얇은 테두리의 칩으로 재구성했습니다. 트림아웃, 플레이스먼트, 깊이, 기타 옵션의 정보 위계가 과하게 튀지 않도록 정리했습니다.
- PREVIEW의 샌드박스 iframe은 결과 확인 전용으로 처리합니다. 따라서 미리보기 영역에서 터치한 뒤에도 앱의 뒤로가기, 사이드바, 오른쪽 퀵메뉴와 엣지 제스처가 차단되지 않습니다.
- 정상적으로 한 번 일치한 결과는 `1개 매치` 대신 **매치**로 표시합니다. 여러 번 일치했을 때만 `N개 매치`로 표시합니다.

## 사용자 지정 컬러 테마

- `옅은 강조·선택 배경(--accent-soft)`을 퀵메뉴의 기본 아이콘 첫 그라데이션 색(`--quickmenu-default-icon-bg-a`)에 연결했습니다.
- 같은 색을 퀵메뉴 메인 패널 하단 그라데이션(`--quickmenu-panel-bg-b`)에도 연결했습니다.
- 기존 사용자 테마는 v12 정규화로 한 번 저장되어, 이미 선택한 옅은 강조색을 그대로 승계합니다.

## 자유 메모

- 툴바의 파일 첨부 바로 오른쪽에 `＋` 삽입 버튼을 추가했습니다.
- **내 글 바로가기**: 제목·프로젝트 검색으로 다른 메모를 골라 현재 커서에 칩 링크를 삽입합니다. 프로젝트 이동 후에도 저장된 메모 ID를 따라가며, 읽기 화면에서 탭하면 대상 메모를 엽니다.
- **구분선**: 커서 위치에 정적 구분선을 넣고 다음 문단에서 이어 쓸 수 있습니다.
- HTML 코드 보기 상태에서는 리치 본문 삽입을 막고, 기존 코드 편집 흐름을 유지합니다.

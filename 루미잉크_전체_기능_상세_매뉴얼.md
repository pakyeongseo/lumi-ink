# 루미잉크 전체 기능 상세 매뉴얼

> 기준 버전: Lumi Ink v59 · 2026-06-23 현재 수정본  
> 문서 성격: 홍보문이 아닌 실제 사용 설명서  
> 대상: 처음 사용하는 이용자, 기존 이용자, 데이터를 이전하려는 이용자, 로그 템플릿 제작자

## 1. 루미잉크란 무엇인가

루미잉크는 창작 자료를 프로젝트 단위로 정리하는 로컬 우선 메모 웹앱이다. 일반 문서를 쓰는 자유 메모뿐 아니라, AI 롤플레이와 세계관 관리에 맞춘 로어북, 대화 로그를 게시용으로 꾸미는 로그 저장, 국문·영문 설정을 함께 기록하는 페르소나, 여러 인물을 한 문서에 묶는 캐릭터 메모를 제공한다.

앱의 기본 구조는 다음과 같다.

1. 가장 바깥에 프로젝트가 있다.
2. 각 프로젝트 안에 다섯 종류의 메모를 만든다.
3. 메모 유형에 따라 편집 화면과 내보내기 형식이 달라진다.
4. 작성 데이터는 계정 서버가 아니라 현재 브라우저의 로컬 저장공간에 보관된다.
5. 다른 기기나 브라우저로 옮길 때는 전체 백업 파일을 사용한다.

루미잉크에는 별도의 로그인이나 클라우드 동기화 기능이 없다. 같은 기기라도 다른 브라우저, 다른 브라우저 프로필, 시크릿 모드, 다른 배포 주소는 서로 다른 저장공간으로 취급될 수 있다. 따라서 중요한 자료는 주기적으로 전체 백업해야 한다.

| 프로젝트 | 작품·세션·세계관별 자료 묶기 | 태그, 색상, 썸네일, 프레임, 고정, 복제 | 재가져오기 가능한 프로젝트 HTML |
| 자유 메모 | 자유로운 문서 작성 | 리치 서식, 이미지, 링크, 코드 블록, 파일 첨부 | 일반 HTML |

## 3. 설치와 실행

### 3.1 웹앱으로 사용하기

배포된 루미잉크 주소를 브라우저에서 열면 별도 설치 없이 사용할 수 있다. 다만 브라우저 데이터 삭제, 사이트 데이터 삭제, 시크릿 모드 종료 등으로 로컬 데이터가 사라질 수 있으므로 장기 사용 전에는 설치형 PWA로 추가하고 백업 습관을 잡는 편이 좋다.

### 3.2 Android

1. Chrome에서 루미잉크 주소를 연다.
2. 브라우저의 점 세 개 메뉴를 연다.
3. `앱 설치` 또는 `홈 화면에 추가`를 선택한다.
4. 설치된 루미잉크 아이콘으로 실행한다.

### 3.3 iPhone·iPad

1. Safari에서 루미잉크 주소를 연다.
2. 공유 버튼을 누른다.
3. `홈 화면에 추가`를 선택한다.

### 3.4 데스크톱

Chrome 또는 Edge의 주소창 설치 아이콘을 누르거나, 브라우저 메뉴에서 앱 설치를 선택한다. 설치 후에는 독립 창 형태로 실행할 수 있다.

### 3.5 설치 아이콘 선택

`설정 → 앱 설치 아이콘`에서 다음 10종을 고를 수 있다.

- 잉크 블루
- 골드
- 바이올렛
- 로즈
- 포레스트
- 파스텔 블루
- 파스텔 핑크
- 파스텔 그린
- 파스텔 퍼플
- 파스텔 옐로우

아이콘을 선택하면 해당 아이콘이 연결된 설치 전용 페이지로 이동한다. 이미 설치한 PWA의 아이콘은 선택만으로 즉시 교체되지 않을 수 있다. 이 경우 기존 설치본을 제거하고 원하는 아이콘 페이지에서 다시 설치해야 한다.

### 3.6 오프라인 사용과 업데이트

핵심 앱 파일, 토크나이저, 아이콘, 로그 템플릿 등은 서비스 워커 캐시에 저장된다. 한 번 정상적으로 불러온 뒤에는 네트워크가 끊겨도 대부분의 작성·열람 기능을 사용할 수 있다.

앱의 HTML과 JavaScript는 온라인일 때 새 파일을 먼저 확인하고, 네트워크를 사용할 수 없을 때 캐시로 돌아간다. 업데이트가 배포되었는데 이전 화면이 남아 있다면 온라인 상태에서 앱을 다시 열거나 새로고침한다.

웹폰트를 외부 주소로 설정한 경우 해당 폰트의 최초 로드는 인터넷 연결이 필요할 수 있다. 폰트를 불러오지 못해도 작성 데이터가 사라지지는 않으며 대체 글꼴로 표시된다.

## 4. 화면과 기본 조작

### 4.1 홈 화면

홈 화면은 프로젝트와 최근 메모를 빠르게 여는 시작 화면이다.

- 왼쪽 상단 메뉴 버튼: 사이드바를 연다.
- 가운데 Lumi Ink 로고: 홈 목록을 맨 위로 부드럽게 이동한다.
- 오른쪽 상단 설정 버튼: 설정 화면을 연다.
- 검색창: 프로젝트와 메모를 통합 검색한다.
- `새 메모`: 메모 유형과 저장할 프로젝트를 순서대로 선택한다.
- `새 프로젝트`: 이름과 태그를 입력해 프로젝트를 만든다.
- 프로젝트 영역: 프로젝트 카드, 메모 수, 수정 시각을 표시한다.
- 최근 수정한 메모: 최대 10개의 최근 메모를 표시한다.
- 하단 통계: 전체 프로젝트 수와 전체 메모 수를 표시한다.
- 우측 하단 `+` 버튼: 새 메모 만들기를 연다.

프로젝트 수가 적어 프로젝트 영역 자체에 스크롤이 생기지 않더라도, 그 영역에서 위아래로 휠이나 스와이프하면 홈 전체 화면의 스크롤로 자연스럽게 이어진다. 프로젝트 목록이 긴 경우에는 목록의 위·아래 경계에서 바깥 홈 스크롤로 연결된다.

### 4.2 사이드바

어느 주요 화면에서든 왼쪽 위 메뉴 버튼을 누르면 사이드바를 열 수 있다. 모바일에서는 화면 왼쪽 가장자리에서 오른쪽으로 스와이프해도 열린다.

사이드바에는 다음 항목이 있다.

- 전체 프로젝트와 각 프로젝트의 메모 수
- 최근 수정한 메모 최대 10개
- 새 메모
- 새 프로젝트
- `열기`: HTML 또는 JSON 파일 가져오기
- 설정
- 밝게/어둡게 빠른 전환

사이드바의 Lumi Ink 로고를 누르면 홈으로 돌아간다.

### 4.3 뒤로 가기와 겹친 창 닫기

브라우저 또는 기기의 뒤로 가기는 먼저 현재 화면 위에 열린 요소를 닫는다. 선택 모드, 이미지 자르기, 이미지 확대 화면, 팝업, 하단 메뉴, 사이드바 순으로 가장 위의 요소를 정리한 뒤 이전 화면으로 이동한다.

작성 화면에서 뒤로 가거나 다른 메모를 열 때에는 현재 편집 내용을 먼저 저장한 뒤 화면을 전환한다.

### 4.4 길게 누르기와 우클릭

프로젝트 카드나 메모 카드를 길게 누르면 해당 항목의 관리 메뉴가 열린다. 데스크톱에서는 우클릭으로 같은 메뉴를 열 수 있다.

로그 템플릿은 길게 누르기의 의미가 다르다. 템플릿 선택 창에서 길게 누르면 템플릿이 즐겨찾기에 추가되거나 해제된다.

### 4.5 저장 상태 표시

각 편집 화면의 상단에는 `기록 중`, `저장됨` 상태가 잠시 표시된다. 일반적인 텍스트 입력은 약 0.55초 동안 추가 입력이 없으면 자동 저장된다. 화면을 떠날 때에도 남아 있는 편집 내용을 저장한다.

상단 저장 버튼을 누르거나 Windows/Linux에서는 `Ctrl+S`, macOS에서는 `⌘+S`를 누르면 즉시 저장할 수 있다. 이 단축키는 자유 메모, 로어북, 로그, 페르소나, 캐릭터 메모 편집 화면에서 동작한다.

## 5. 프로젝트 관리

### 5.1 프로젝트 만들기

처음 실행하면 삭제할 수 없는 `기본 메모함`이 자동으로 준비된다. 프로젝트를 따로 나누고 싶지 않은 자료나 가져오기 과정에서 위치를 잃은 메모를 안전하게 모으는 기본 위치다.

1. 홈이나 사이드바에서 `새 프로젝트`를 누른다.
2. 프로젝트 이름을 입력한다. 이름은 필수이며 최대 60자다.
3. 선택적으로 태그를 쉼표로 구분해 입력한다. 입력란 전체는 최대 500자다.
4. `만들기`를 누른다.

프로젝트 편집 창에는 과거 호환을 위해 내부적으로 설명 필드가 남아 있지만, 현재 화면에서는 쉼표로 나눈 값을 프로젝트 태그 칩으로 표시한다. 예를 들어 `판타지, 로맨스, 진행중`을 입력하면 프로젝트 상세 상단에 세 개의 태그가 나타난다.

### 5.2 프로젝트 상세 화면

프로젝트를 열면 다음 정보가 보인다.

- 프로젝트 썸네일
- 프로젝트 이름
- 전체 메모 수
- 프로젝트 태그
- 유형별 메모 목록

메모 섹션은 `캐릭터 → 페르소나 → 로그 → 로어북 → 자유 메모` 순서로 표시된다. 각 섹션 안에서는 먼저 고정된 메모가 나오고, 나머지 메모가 선택한 정렬 방식으로 배치된다.

### 5.3 프로젝트 정렬

홈의 정렬 버튼에서 다음 네 가지를 선택할 수 있다.

- 최신순
- 오래된순
- 이름순 ㄱ→ㅎ
- 이름순 ㅎ→ㄱ

프로젝트 상세 화면의 첫 메모 섹션에 있는 정렬 버튼은 모든 메모 섹션의 정렬 기준을 함께 바꾼다. 홈 프로젝트 정렬과 프로젝트 안 메모 정렬은 서로 별도로 저장된다.

### 5.4 프로젝트 관리 메뉴

프로젝트 카드의 점 세 개 버튼을 누르거나 카드를 길게 누르면 다음 작업을 할 수 있다.

- 새 메모 만들기
- 상단 고정 또는 고정 해제
- 이름·태그 편집
- 프로젝트 색상 지정
- 프로젝트 파일로 내보내기
- 프로젝트 전체 복제
- 프로젝트 아이콘 변경
- 썸네일 프레임 선택
- 소속 로어북을 World Info로 묶어 내보내기
- 프로젝트 삭제

홈 화면에서는 `선택` 항목도 표시되어 여러 프로젝트를 한꺼번에 삭제할 수 있다.

### 5.5 프로젝트 고정

프로젝트는 최대 3개까지 상단에 고정할 수 있다. 고정된 프로젝트끼리는 고정한 순서를 기준으로 표시되고, 아래에 나머지 프로젝트가 이어진다.

### 5.6 프로젝트 색상

프로젝트 카드의 경계와 상세 썸네일 강조에 사용할 색상을 지정할 수 있다. 제공되는 색상은 다음 10종이다.

- 루미 핑크
- 코랄 잉크
- 선셋 오렌지
- 골드 글로우
- 포레스트 그린
- 아쿠아 민트
- 네온 시안
- 미드나잇 블루
- 라벤더 퍼플
- 미스티 그레이

색상 없음으로 되돌리는 항목도 있다.

### 5.7 프로젝트 썸네일

기본 제공 썸네일 16종 중 하나를 고르거나 사용자 이미지를 직접 업로드할 수 있다.

기본 제공 이미지는 `별 고래`, `분홍 창가`, `황금 옥좌`, `얼음 성`, `빛의 숲길`, `야경 도시`, `푸른 하늘성`, `분홍 공주방`, `깃펜과 편지`, `수정 첨탑`, `보라 수정`, `달빛 서재`, `이상한 나라`, `겨울 보름달`, `라벤더 정원`, `산호 바다`다.

업로드 이미지는 정사각형 자르기 화면에서 위치와 확대 비율을 조절한 뒤 512×512 이미지로 저장한다.

### 5.8 썸네일 프레임

프로젝트 썸네일 바깥에는 20종의 장식 프레임을 적용할 수 있다.

`클래식`, `비드`, `바로크`, `아르데코`, `로프`, `고딕`, `보태니컬`, `스캘럽`, `스타라이트`, `로열`, `슬림 헤어라인`, `슬림 코너 브래킷`, `슬림 인셋 닷`, `슬림 파인 비드`, `슬림 다이아 코너`, `슬림 미니 노치`, `슬림 덩굴`, `슬림 물결`, `슬림 점선`, `슬림 스파클`

프레임을 고른 뒤 23종의 고정 색상과 `테마와 연동`을 포함한 총 24가지 표현 중 하나를 선택한다. 고정 색상에는 골드, 로즈골드, 실버, 브론즈, 화이트, 블랙, 네이비, 버건디, 에메랄드, 바이올렛, 스카이, 로즈, 여러 파스텔 계열, 투명 글래스, 글래시한 명암이 포함된다.

- `테마와 연동`: 앱의 컬러 테마를 바꾸면 프레임 색도 함께 바뀐다.
- `투명 글래스`: 반투명 흰색 유리 프레임처럼 표시된다.
- `글래시한 명암`: 반투명 하이라이트와 그림자를 사용해 깊이를 표현한다.

프레임은 홈 카드, 사이드바, 프로젝트 선택 창, 프로젝트 상세 헤더에서 같은 구성으로 표시된다.

### 5.9 프로젝트 복제와 삭제

프로젝트 복제는 프로젝트 정보뿐 아니라 내부의 모든 메모와 자유 메모 첨부파일을 새 ID로 복사한다. 복제본 이름에는 `(사본)`이 붙는다.

프로젝트를 삭제하면 소속 메모와 그 첨부파일도 함께 삭제된다. 삭제 직후 약 6초 동안 표시되는 `되돌리기`를 누르면 프로젝트, 메모, 첨부파일을 복원할 수 있다. 삭제 전에는 자동 백업도 한 번 실행한다.

### 5.10 프로젝트 단위 내보내기와 불러오기

프로젝트 관리 메뉴의 `프로젝트 파일로 내보내기`를 누르면 해당 프로젝트 하나를 독립된 HTML 패키지로 저장한다. 파일에는 다음 데이터가 포함된다.

- 프로젝트 이름, 태그, 색상, 썸네일, 프레임
- 프로젝트 안의 자유 메모, 로어북, 로그, 페르소나, 캐릭터 메모 전체
- 메모 본문에 저장된 이미지
- 자유 메모에 연결한 첨부파일
- 로그에 저장된 사용자 템플릿 스냅숏과 이름 가림 설정

내보낸 HTML은 브라우저에서 열면 프로젝트 이름과 메모 목록을 확인할 수 있으며, 실제 복원 데이터도 파일 안에 함께 들어 있다. 다른 루미잉크에서 가져오려면 사이드바 하단의 `파일 열기`를 누르고 이 HTML을 선택한 뒤 `새 프로젝트로 가져오기`를 누른다.

불러오기는 기존 프로젝트를 덮어쓰거나 병합하지 않는다. 프로젝트·메모·첨부파일에 모두 새 ID를 발급해 별개의 프로젝트 사본으로 추가한다. 같은 이름의 프로젝트가 이미 있으면 `(가져옴)`, `(가져옴 2)`처럼 이름을 자동 조정한다. 프로젝트 내부 메모 고정은 유지하지만 홈과 사이드바의 전역 고정은 해제해 기존 고정 목록을 침범하지 않는다.

이 기능은 한 프로젝트를 전달하거나 다른 작업 공간으로 옮기는 용도다. 모든 프로젝트와 앱 전체 상태를 보존하려면 `설정 → 전체 백업`을 사용한다.

## 6. 메모 만들기와 공통 관리

### 6.1 새 메모 만들기

1. 홈·사이드바·프로젝트 상세의 `새 메모` 또는 `+` 버튼을 누른다.
2. 자유 메모, 로어북, 로그 저장, 페르소나, 캐릭터 메모 중 하나를 고른다.
3. 프로젝트 안에서 시작했다면 그 프로젝트에 바로 생성된다.
4. 홈이나 사이드바에서 시작했다면 저장할 프로젝트를 선택한다.
5. 프로젝트가 없다면 선택 과정에서 새 프로젝트를 만들 수 있다.

### 6.2 자동 제목과 수동 제목

| 메모 유형 | 자동 제목 규칙 |
|---|---|
| 자유 메모 | 본문의 첫 번째 비어 있지 않은 줄을 최대 52자로 사용 |
| 로어북 | 기본값 `이름 없는 로어북`; 필요할 때 직접 변경 |
| 로그 | 원문 첫 줄에서 따옴표·대괄호·별표 등의 표식 일부를 제거해 최대 52자로 사용 |
| 페르소나 | 한국어 이름, 없으면 영어 이름 사용 |
| 캐릭터 메모 | 첫 캐릭터 이름을 사용하고 여러 페이지면 `이름 외 N명` 형식 사용 |

메모 관리 메뉴에서 이름을 직접 바꾸면 해당 제목은 잠금 상태가 되어 이후 본문이나 인물 이름이 바뀌어도 자동 제목으로 덮어쓰지 않는다.

### 6.3 메모별 색상

각 메모에는 프로젝트 색상과 별개로 10종의 색상 표식을 지정할 수 있다. 자유 메모·로어북·로그는 목록의 점과 경계에 반영되고, 페르소나·캐릭터는 썸네일 카드 경계에 반영된다.

### 6.4 메모 고정

프로젝트 상세의 각 메모 유형 섹션마다 최대 3개를 고정할 수 있다. 예를 들어 자유 메모 3개와 로그 3개를 각각 고정할 수 있다.

홈의 최근 메모와 사이드바의 최근 메모에는 서로 독립적인 고정 상태가 있다. 각 위치에서 최대 3개까지 상단에 고정할 수 있다. 홈 고정과 사이드바 고정은 프로젝트 상세의 메모 고정과 별개다.

### 6.5 메모 관리 메뉴의 공통 작업

메모 유형에 따라 세부 항목은 다르지만 일반적으로 다음 기능을 제공한다.

- 선택 모드 진입
- 상단 고정 또는 해제
- 이름 바꾸기
- 색상 지정
- 해당 유형의 파일로 내보내기
- 다른 프로젝트로 이동
- 선택한 프로젝트에 복제
- 삭제

복제할 때 자유 메모 첨부파일도 새 파일 레코드로 함께 복사된다. 삭제 직후에는 약 6초 동안 되돌릴 수 있다.

## 7. 자유 메모

### 7.1 용도와 화면 구분

자유 메모는 HTML 서식을 보존하는 일반 문서형 메모다. 처음 열면 읽기 화면이 나타나며, 상단 편집 버튼이나 본문 더블클릭으로 편집 화면에 들어간다.

- 읽기 화면: 완성된 본문과 첨부파일을 열람한다.
- 편집 화면: 리치 에디터 또는 HTML 소스로 내용을 수정한다.

읽기 화면의 본문을 길게 누르면 서식을 제외한 본문 전체 텍스트를 클립보드에 복사한다.

### 7.2 서식 도구

편집 툴바에서 다음 기능을 사용할 수 있다.

- 굵게
- 기울임
- 밑줄
- 취소선
- 글자 크기 한 단계 줄이기·늘리기
- 글자 크기 목록 선택
- 글자 색
- 형광펜
- 왼쪽·가운데·오른쪽 정렬
- 글머리표 목록
- 번호 목록
- 이미지 삽입
- 일반 파일 첨부
- 하이퍼링크
- 코드 블록
- 서식·코드 블록·링크 지우기
- 실행 취소
- HTML 소스 보기·편집
- 툴바 접기·펼치기

### 7.3 글자 크기

텍스트를 선택한 뒤 크기 목록에서 다음 일곱 단계 중 하나를 고를 수 있다.

`아주 작게 12px`, `작게 14px`, `보통 16px`, `크게 19px`, `더 크게 23px`, `제목 28px`, `대제목 34px`

`A−`, `A+`는 현재 크기를 한 단계씩 바꾼다.

### 7.4 글자 색

기본 팔레트, 운영체제 색상 선택기, HEX 직접 입력을 지원한다. 마음에 드는 색을 `내 색상`에 저장할 수 있으며 최대 24개까지 유지한다. 마지막으로 적용한 색도 다음 편집 때 기억한다.

### 7.5 형광펜

텍스트를 선택한 뒤 형광펜 버튼을 짧게 누르면 현재 형광펜 색을 적용하거나 이미 적용된 형광펜을 제거한다. 버튼을 약 0.48초 동안 길게 누르면 다음 색상 선택 창이 열린다.

- 하늘색
- 핑크색
- 연두색
- 연보라색
- 노란색

선택한 형광펜 색은 다음 사용에도 유지되며 캐릭터 크리에이터 메모의 형광펜과 공유된다.

### 7.6 HTML 소스 편집

`<>` 형태의 코드 버튼을 누르면 현재 메모의 HTML 소스를 직접 볼 수 있다. 소스 모드에서 수정한 HTML은 일반 편집 모드와 같은 본문 데이터로 저장된다.

이 기능은 고급 사용자용이다. 앱 안에서 직접 입력한 소스는 가져온 외부 HTML과 달리 별도의 전체 정화 과정을 거치지 않으므로, 출처를 알 수 없는 HTML·이벤트 속성·스크립트성 코드를 붙여넣지 않는 것이 안전하다.

### 7.7 이미지 삽입

이미지는 문서 본문 안에 삽입된다. 긴 변을 최대 1280px로 줄이고 JPEG 데이터로 변환한 뒤 메모 HTML 안에 포함한다.

이미지 입력 제한은 다음과 같다.

- 파일 크기 12MB 이하
- 전체 해상도 2,400만 픽셀 이하
- 일반 이미지 파일만 허용
- SVG는 업로드 이미지로 허용하지 않음

### 7.8 파일 첨부

본문 이미지와 별개로 일반 파일을 첨부할 수 있다.

- 한 파일당 최대 15MB
- 이미지, 문서, 압축 파일, 오디오, 비디오 등 파일 유형에 맞춘 아이콘 표시
- 읽기·편집 화면에서 파일 다운로드 가능
- 편집 화면에서 첨부 삭제 가능
- 첨부 삭제는 확인 창을 거치지만 즉시 되돌리기 버튼은 제공하지 않음

첨부파일은 메모 본문이 아니라 브라우저의 별도 파일 저장소에 보관된다. 프로젝트·메모 복제와 전체 백업에는 포함되지만, 자유 메모 하나를 일반 HTML로 저장할 때에는 첨부파일이 포함되지 않는다.

### 7.9 링크

링크를 걸 텍스트를 먼저 선택한 뒤 링크 버튼을 누르고 주소를 입력한다. `https://`를 생략하면 자동으로 붙인다.

또한 다음 동작을 지원한다.

- `http://` 또는 `https://` 주소를 입력한 뒤 공백이나 Enter를 누르면 자동 링크 변환
- 선택한 텍스트 위에 URL만 붙여넣으면 해당 텍스트에 링크 적용
- 선택 영역 없이 URL만 붙여넣으면 URL 자체를 링크로 삽입
- 읽기 화면의 링크는 새 창에서 열리고 `noopener noreferrer` 보호 속성을 사용

### 7.10 코드 블록

코드 블록 버튼은 현재 문단을 `pre` 형식으로 바꾼다. 읽기 화면에서는 각 코드 블록 오른쪽 위에 `복사` 버튼이 자동으로 붙는다.

지우개 버튼은 선택 영역의 일반 서식, 링크, 코드 블록·코드 요소를 함께 제거한다.

### 7.11 자유 메모 HTML 저장과 다시 열기

메모 관리 메뉴의 `.html로 저장`은 본문 서식과 본문 안 이미지가 포함된 일반 HTML 문서를 만든다. 이 파일은 브라우저에서 독립적으로 열 수 있다.

주의할 점은 다음과 같다.

- 별도 첨부파일은 포함되지 않는다.
- 루미잉크 고유 메타데이터를 넣는 페르소나·캐릭터·로그 HTML과 달리, 자유 메모 HTML은 일반 문서 형식이다.
- 루미잉크의 `열기`로 다시 가져오면 HTML 제목과 정화된 본문을 사용해 새 자유 메모를 만든다.
- 외부 HTML을 가져올 때 스크립트, 폼, iframe, 위험한 속성, 원격 이미지 등은 제거하거나 제한한다.

## 8. 로어북

### 8.1 용도

로어북은 SillyTavern World Info 항목을 작성하고 관리하는 메모 유형이다. 원문은 경량 마크다운으로 저장하며, 발동 키워드와 constant, 삽입 깊이를 함께 기록한다.

### 8.2 발동 키워드

상단 `발동 키워드` 입력란에 값을 입력한 뒤 Enter 또는 쉼표를 누르면 키워드 칩이 생성된다.

- 한 번에 쉼표로 여러 키워드 추가 가능
- 동일한 문자열의 중복 추가 방지
- 각 칩의 `×` 버튼으로 개별 삭제
- World Info 내보내기 때 `key` 배열로 전달

### 8.3 항상 활성

`항상 활성 (constant)`을 켜면 키워드 일치 여부와 관계없이 항상 활성화되는 World Info 항목으로 내보낸다. JSON에서는 `constant: true`가 된다.

### 8.4 깊이

깊이 스위치를 켜면 World Info의 Depth 위치를 사용한다.

- 입력 범위: 0~999
- 기본값: 4
- 켜짐: 내보내기 JSON의 `position`이 4가 되고 입력한 `depth`를 사용
- 꺼짐: 일반 위치 값으로 내보냄

### 8.5 지원하는 경량 마크다운

| 입력 | 보기 결과 |
|---|---|
| `# 제목`, `## 제목`, `### 제목` | 1~3단계 제목 |
| `**굵게**` | 굵은 글자 |
| `*기울임*` | 기울임 |
| `` `코드` `` | 인라인 코드 |
| `[이름](https://주소)` | 외부 링크 |
| 줄 단독 `@@명령` | 명령 칩 행 |
| 문장 안 `@@명령` | 인라인 명령 칩 |
| `- 항목`, `* 항목`, `+ 항목` | 글머리표 목록 |
| `1. 항목` | 번호 목록 |
| `- 라벨: 값` | 라벨 미니 칩 행 |
| `> 인용` | 인용문 |
| `---`, `***`, `___` | 구분선 |
| 삼중 백틱 | 여러 줄 코드 블록 |
| `<태그>`부터 `</태그>` | 태그 이름이 붙은 박스 |

제목과 목록은 `#제목`, `-내용`처럼 표식 뒤 공백이 없어도 인식한다.

### 8.6 편집과 미리보기

눈 모양 버튼으로 원문 편집과 렌더링된 미리보기를 전환한다. 미리보기 화면을 길게 누르면 꾸며진 화면이 아니라 원본 마크다운 전체를 클립보드에 복사한다.

### 8.7 토큰 계산

로어북 본문은 번들된 cl100k 계열 토크나이저로 토큰 수를 계산한다. 토크나이저는 필요할 때 불러오며 오프라인 캐시 대상이다. 계산할 수 없는 환경에서는 `토큰 계산 불가`로 표시한다.

### 8.8 World Info JSON 내보내기

개별 로어북의 관리 메뉴에서 `World Info(.json)로 내보내기`를 선택할 수 있다. 프로젝트 관리 메뉴에서는 프로젝트 안의 여러 로어북을 선택해 하나의 JSON으로 묶을 수 있다.

여러 항목 내보내기 과정은 다음과 같다.

1. 포함할 로어북을 체크한다.
2. 선택된 항목 수와 합산 토큰을 확인한다.
3. 파일 이름을 지정한다.
4. 하나의 `entries` 객체를 가진 SillyTavern World Info JSON을 저장한다.

각 항목에는 제목, 키워드, 본문, constant, Depth, 기본 확률과 순서값 등이 들어간다.

### 8.9 World Info JSON 가져오기

사이드바의 `열기`에서 World Info JSON을 선택하면 `entries` 객체 또는 항목 배열을 읽는다. 대상 프로젝트를 고르면 각 World Info 항목이 별도의 로어북 메모로 생성된다.

- `comment` → 로어북 제목
- `key` 또는 `keys` → 발동 키워드
- `content` → 본문
- `constant` → 항상 활성
- `position: 4` → 깊이 사용
- `depth` → 깊이 값

### 8.10 로어북 합치기

같은 프로젝트에서 로어북을 두 개 이상 선택한 뒤 `합치기`를 누르면 원본을 삭제하지 않고 `합친 로어북`이라는 새 메모를 만든다. 본문은 빈 줄 두 개로 이어 붙인다. 키워드, constant, Depth는 자동 병합하지 않으므로 새 메모에서 다시 설정해야 한다.

## 9. 로그 저장

### 9.1 기본 개념

로그 저장은 원문과 꾸며진 보기를 분리하는 메모 유형이다.

- 편집 모드: 사용자가 입력한 원문을 그대로 수정한다.
- 보기 모드: 선택한 템플릿의 정규식과 스타일을 적용한 결과만 보여준다.
- 원문에는 꾸밈용 HTML을 직접 삽입하지 않는다.
- 템플릿을 바꾸거나 이름 가림을 적용해도 원문은 유지된다.

눈·연필 버튼으로 두 모드를 전환한다.

### 9.2 정규식 디자인

템플릿은 일반적으로 다음 표식을 찾아 서로 다른 스타일을 적용한다.

- `"큰따옴표 안쪽"`
- `'작은따옴표 안쪽'`
- `[대괄호 안쪽]`
- `**굵게 형식**`
- `*기울임 형식*`

실제 인식 표식, 구분자 제거 여부, 색상과 테두리는 템플릿마다 다르다. 규칙은 템플릿 JSON의 배열 순서대로 적용된다.

### 9.3 이름 가림

로그에 등장하는 페르소나명, 본명, 애칭, 호칭 등을 보기 결과에서 다른 표시로 치환할 수 있다.

1. `가릴 원본 이름`에 이름을 입력한다.
2. 추가 이름이 있으면 `+ 원본 이름`을 누른다.
3. 필요하면 `대체 표시`를 입력한다.
4. `이름 가림 적용`을 누른다.

세부 동작은 다음과 같다.

- 원본 이름은 최대 20개다.
- 같은 이름은 데이터 저장 시 중복 제거된다.
- 여러 이름이 겹치면 긴 이름부터 처리한다.
- 이름 비교는 대소문자를 구분하는 정확한 문자열 검색이다.
- 단어 경계를 따로 보지 않으므로 다른 단어 안에 포함된 문자열도 치환될 수 있다.
- 대체 표시를 비워 두면 템플릿이 지정한 기본 가림 문자열을 사용한다.
- 모든 이름은 같은 대체 문자열과 같은 템플릿 가림 스타일을 사용한다.
- 이름 가림은 본문뿐 아니라 꾸며진 로그의 제목 영역에도 적용된다.
- 원문과 메모의 실제 제목 데이터는 바뀌지 않고 보기·복사·내보내기 결과에서만 가린다.

### 9.4 기본 제공 템플릿 50종

템플릿 선택 창의 `기본` 탭에는 50종이 들어 있다. 목록은 이름순으로 정렬할 수 있으며, 별도의 `50종 소개 보기` 페이지에서 샘플과 설명을 확인할 수 있다.

전체 이름은 이 문서의 부록 A에 정리되어 있다.

### 9.5 템플릿 선택 창

템플릿 창은 세 탭으로 구성된다.

- `기본`: 앱에 포함된 50종
- `사용자`: 사용자가 올린 템플릿과 현재 메모에 보존된 사용자 템플릿
- `즐겨찾기`: 길게 눌러 고정한 템플릿

창 안에서 할 수 있는 작업은 다음과 같다.

- 템플릿 이름, 설명, 제작자, ID, 규칙 이름, 정규식 내용 검색
- 이름 오름차순·내림차순 정렬
- 템플릿을 길게 눌러 즐겨찾기 추가·해제
- 즐겨찾기된 템플릿을 일반 탭 상단에 우선 표시
- 사용자 템플릿 삭제
- JSON 템플릿 업로드
- 제작 가이드 다운로드
- 50종 소개 페이지 열기

창을 닫았다가 다시 열면 `기본 탭`, `빈 검색어`, `이름 오름차순`으로 초기화된다. 사용자 템플릿과 즐겨찾기 정보만 지속적으로 보존된다.

### 9.6 사용자 템플릿

템플릿 선택 창의 `JSON 템플릿 업로드`로 사용자 템플릿을 추가한다.

주요 제한은 다음과 같다.

- UTF-8 JSON 파일
- 파일 크기 200KB 이하
- `kind: lumink-log-template`
- `schemaVersion: 1`
- ID는 영문·숫자·하이픈으로 이루어진 3~64자
- 기본 템플릿과 ID 중복 불가
- 사용자 템플릿 최대 20개
- 정규식 규칙 최대 20개
- 허용된 CSS 속성만 사용
- 외부 URL, `@import`, `expression`, `javascript:` 등 위험한 스타일 차단
- 지나치게 복잡하거나 위험한 정규식 거부

사용자 템플릿을 메모에 적용하면 해당 템플릿의 스냅샷이 메모 안에도 저장된다. 나중에 사용자 템플릿 목록에서 삭제해도 이미 적용된 메모는 보존된 스냅샷으로 계속 렌더링할 수 있다.

전체 제작 규격은 [로그 템플릿 제작 가이드](./lumink-log-template-guide.md)를 참고한다.

### 9.7 인라인 HTML 코드 복사

메모 관리 메뉴의 `인라인 HTML 코드 복사`는 현재 템플릿 결과를 하나의 HTML 조각으로 만든다.

- 배경, 테두리, 글자색, 문단 스타일을 모두 `style` 속성에 인라인 처리
- 외부 스타일시트 없이 게시글 HTML 영역에 붙여넣기 쉬운 형태
- 현재 제목과 본문에 이름 가림 적용
- 원본 로그 데이터나 편집 UI는 포함하지 않음

게시판이 인라인 스타일을 제거하거나 허용 HTML을 제한하면 일부 디자인은 달라질 수 있다.

### 9.8 꾸며진 HTML 파일 저장

`꾸며진 HTML 파일로 저장`은 브라우저에서 독립적으로 열 수 있는 완성 문서를 만든다. 화면에 보이는 인라인 디자인과 함께 루미잉크 재가져오기용 로그 데이터도 문서 안에 포함한다.

이 파일을 사이드바의 `열기`로 가져오면 대상 프로젝트에 새 로그 메모가 만들어지고, 원문·템플릿·이름 가림 설정을 다시 편집할 수 있다.

### 9.9 로그 합치기

같은 프로젝트의 로그를 두 개 이상 선택해 합치면 원본은 그대로 두고 `합친 로그`를 새로 만든다.

- 본문은 빈 줄 두 개로 연결한다.
- 템플릿과 이름 가림 설정은 선택된 첫 번째 로그의 설정을 사용한다.
- 새 로그의 제목은 `합친 로그`로 잠긴다.
- 결과는 보기 모드로 열린다.

## 10. 페르소나

### 10.1 편집 모드와 보기 모드

페르소나는 한 인물의 이미지와 설정을 카드 형태로 관리한다. 상단 눈·연필 버튼으로 편집과 보기를 전환한다.

### 10.2 한국어·영어 이중 양식

한국어와 English 탭은 각각 다음 데이터를 따로 가진다.

- 이름
- 태그
- 상세 설명
- 상세 설명 토큰 수

이미지와 갤러리는 두 언어 탭이 공유한다. 한국어 이름이 비어 있으면 영어 이름이 메모 자동 제목에 사용된다.

### 10.3 포트레이트와 정사각 썸네일

- 포트레이트: 3:4 비율, 1200×1600으로 저장
- 정사각 썸네일: 1:1 비율, 1080×1080으로 저장

이미지를 선택하면 자르기 화면이 열린다. 이미지를 드래그해 위치를 옮기고 확대 슬라이더로 배율을 조절한 뒤 적용한다. 정사각 썸네일이 없으면 보기 화면과 목록에서 포트레이트를 대신 사용한다.

이미지 제한은 자유 메모와 동일하게 12MB 이하, 2,400만 픽셀 이하다.

### 10.4 태그

쉼표 또는 Enter로 여러 태그를 추가할 수 있다. 중복 태그는 다시 추가하지 않으며 각 태그의 `×` 버튼으로 삭제한다.

### 10.5 상세 설명 문법

페르소나와 캐릭터의 상세 설명은 전체 마크다운이 아니라 카드용 경량 구조 문법을 사용한다.

| 입력 | 보기 결과 |
|---|---|
| `# 제목`부터 `###### 제목` | 제목 박스; 화면에서는 최대 3단계 크기로 정리 |
| `- 라벨: 값` | 라벨 미니 칩 |
| `- 항목` | 삼각형 글머리표 |
| 빈 줄 | 문단 간격 |
| 일반 줄 | 줄바꿈을 유지한 본문 |

`<user name="{{user}}">`처럼 사용자 제어를 위해 넣은 `<user ...>`와 `</user>` 표시는 보기 화면에서 숨긴다. 태그 안쪽의 설정 내용은 유지된다.

### 10.6 토큰 수

한국어와 영어 상세 설명의 토큰을 각각 계산한다. 보기 모드에서는 현재 선택한 언어의 토큰을 표시한다.

### 10.7 갤러리

갤러리의 `+` 버튼에서 여러 이미지를 한 번에 선택할 수 있다. 각 이미지는 긴 변 최대 1600px로 줄여 저장한다.

- 이미지를 약 0.3초 누른 뒤 다른 위치로 끌어 순서 변경
- 삭제 후 약 6초 동안 되돌리기 가능
- 보기 모드에서 이미지를 눌러 확대
- 확대 화면의 이전·다음 버튼, 모바일 좌우 스와이프, 키보드 좌우 화살표 지원
- 확대 화면에서 인접 이미지를 미리 불러와 전환 지연 감소

### 10.8 빠른 복사

보기 모드의 상세 설명을 길게 누르면 현재 언어의 원본 상세 설명을 클립보드에 복사한다.

### 10.9 페르소나 HTML 내보내기

관리 메뉴에서 `HTML로 저장`을 누르면 밝게 또는 어둡게를 선택한다. 앱의 현재 컬러 테마 기조를 유지하면서 카드 전체의 밝기만 결정한다.

내보낸 HTML에는 다음이 포함된다.

- 메모 제목
- 포트레이트와 정사각 썸네일
- 한국어·영어 이름, 태그, 상세 설명
- 갤러리
- 재가져오기용 페르소나 데이터

사이드바의 `열기`로 이 HTML을 선택하면 편집 가능한 새 페르소나 메모로 복원된다.

## 11. 캐릭터 메모

### 11.1 페르소나와의 차이

페르소나는 한 메모에 한 인물을 저장한다. 캐릭터 메모는 한 메모 안에 여러 캐릭터 페이지를 만들고, 각 페이지마다 페르소나 수준의 국문·영문 설정과 이미지, 제작 메모를 보관한다.

### 11.2 캐릭터 페이지

상단 캐릭터 네비게이션 옆 `+` 버튼으로 새 페이지를 추가한다. 페이지마다 다음 데이터가 독립적이다.

- 포트레이트
- 정사각 썸네일
- 갤러리
- 한국어 이름·태그·상세 설명
- 영어 이름·태그·상세 설명
- 크리에이터 메모

캐릭터 페이지가 두 개 이상이면 관리 메뉴에서 현재 페이지를 삭제할 수 있다. 최소 한 페이지는 항상 남아 있어야 한다. 페이지 삭제 직후 약 6초 동안 되돌릴 수 있다.

### 11.3 페이지 이동

- 상단 썸네일 네비게이션에서 인물 선택
- 보기 모드 하단의 이전·다음 화살표
- 모바일 보기 모드에서 좌우 스와이프

편집 중 페이지를 바꾸면 현재 페이지를 먼저 저장한다.

### 11.4 국문·영문 설정과 이미지

각 페이지의 한국어·영어 필드, 태그, 토큰 계산, 상세 설명 문법은 페르소나와 같다. 포트레이트는 3:4, 정사각 썸네일은 1:1 자르기를 사용한다.

갤러리에서는 여러 이미지를 한 번에 추가하고, 보기 모드에서 확대·이전·다음·스와이프를 사용할 수 있다. 현재 캐릭터 갤러리는 페르소나 갤러리와 달리 드래그 순서 변경 기능을 제공하지 않는다.

### 11.5 크리에이터 메모

크리에이터 메모는 공개용 캐릭터 설명과 별도로 제작 메모, 연출 주의점, 관계성 설계 등을 기록하는 리치 텍스트 영역이다.

지원 도구는 다음과 같다.

- HTML 소스 보기·편집
- 굵게, 기울임, 밑줄, 취소선
- 일곱 단계 글자 크기
- 글자 색
- 다섯 색상 형광펜
- 왼쪽·가운데·오른쪽 정렬
- 글머리표·번호 목록
- 이미지 삽입
- 하이퍼링크와 URL 자동 링크
- 코드 블록
- 서식·코드·링크 지우기
- 실행 취소

보기 모드에서는 크리에이터 메모가 있을 때 `CREATOR NOTE` 영역이 나타나며 `크게 보기`를 눌러 팝업으로 열 수 있다. 팝업의 코드 블록에는 복사 버튼이 붙는다.

### 11.6 대표 썸네일

프로젝트 목록의 캐릭터 메모 카드에 사용할 대표 이미지를 별도로 지정할 수 있다.

- 자동 선택: 첫 캐릭터 페이지의 정사각 썸네일, 포트레이트, 첫 갤러리 이미지 순으로 사용
- 기존 캐릭터 페이지의 이미지 중 선택
- 캐릭터 페이지와 별개인 새 대표 이미지 직접 업로드
- 직접 업로드 이미지는 1:1로 잘라 1080×1080으로 저장

대표 썸네일은 프로젝트 카드 안에서 이 캐릭터 메모를 식별하기 위한 표지이며, 개별 캐릭터 페이지 이미지와는 별도다.

### 11.7 자동 제목

수동으로 메모 이름을 바꾸지 않은 경우 첫 캐릭터의 이름을 사용한다. 페이지가 여러 개면 `첫 캐릭터 이름 외 N명`으로 표시한다.

### 11.8 캐릭터 HTML 내보내기

관리 메뉴의 `HTML로 저장`에서 다음을 선택한다.

- 밝게 또는 어둡게
- 크리에이터 메모 포함 또는 제외

크리에이터 메모의 기본값은 `미포함`이다. 공유용 카드에 비공개 제작 메모가 실수로 섞이는 것을 줄이기 위한 설정이다.

내보낸 파일은 여러 캐릭터 페이지를 전환하는 네비게이션을 포함하며, 루미잉크 재가져오기용 데이터도 내장한다. 크리에이터 메모를 제외했다면 재가져오기 데이터에서도 해당 메모가 비어 있는 상태로 저장된다.

## 12. 통합 검색

검색 화면은 `제목만`, `내용만`, `제목+내용` 세 범위를 지원한다. 입력하는 즉시 결과를 갱신하며 프로젝트와 메모 결과를 따로 표시한다.

### 12.1 프로젝트 검색 범위

- 제목: 프로젝트 이름
- 내용: 프로젝트 태그 입력란의 전체 문자열

### 12.2 메모 유형별 내용 검색 범위

| 유형 | 검색되는 내용 |
|---|---|
| 자유 메모 | HTML 태그를 제외한 본문 텍스트 |
| 로어북 | 원문과 발동 키워드 |
| 로그 | 원문, 가릴 원본 이름들, 대체 표시 |
| 페르소나 | 국문·영문 이름, 태그, 상세 설명 |
| 캐릭터 | 모든 페이지의 국문·영문 이름, 태그, 상세 설명, 크리에이터 메모 |

메모 검색 결과는 최근 수정 순으로 표시된다. 검색 결과의 카드도 일반 목록과 마찬가지로 열거나 길게 눌러 관리할 수 있다.

## 13. 다중 선택·이동·합치기

### 13.1 선택 모드 진입

- 홈의 프로젝트 카드 관리 메뉴에서 `선택`
- 프로젝트 상세 상단의 선택 버튼
- 메모 관리 메뉴의 `선택`

선택 모드 하단 바에서 선택 개수, 전체 선택, 합치기, 이동, 삭제를 사용할 수 있다.

### 13.2 여러 메모 이동

같은 프로젝트에서 선택한 메모들을 다른 프로젝트로 한꺼번에 이동한다. 새 프로젝트를 만든 뒤 그 위치를 바로 선택할 수도 있다.

### 13.3 합치기 조건

- 두 개 이상 선택해야 한다.
- 같은 메모 유형끼리만 합칠 수 있다.
- 페르소나와 캐릭터 메모는 합칠 수 없다.
- 합치기는 원본을 삭제하지 않고 새 메모를 만든다.

### 13.4 유형별 합치기 결과

- 자유 메모: HTML 본문 사이에 빈 줄을 넣어 연결하고 첨부파일도 새 ID로 복제한다.
- 로어북: 원문만 빈 줄 두 개로 연결한다. 키워드와 활성·깊이 설정은 초기값이다.
- 로그: 원문을 연결하고 첫 번째 선택 로그의 템플릿·이름 가림 설정을 사용한다.

### 13.5 일괄 삭제

프로젝트 일괄 삭제는 각 프로젝트의 소속 메모와 첨부파일도 함께 삭제한다. 메모 일괄 삭제는 선택한 메모와 연결 첨부파일을 삭제한다. 삭제 후 약 6초 동안 한 번의 되돌리기 작업을 제공한다.

## 14. 파일 열기와 형식 판별

사이드바 하단의 `파일 열기`는 `.html`, `.htm`, `.json` 파일을 받는다. 파일 내용에 따라 다음처럼 처리한다.

| 파일 | 처리 방식 |
|---|---|
| World Info JSON | 항목별 로어북으로 가져오기 |
| 루미잉크 프로젝트 HTML | 메모와 첨부파일을 포함한 새 프로젝트로 복원 |
| 루미잉크 페르소나 HTML | 편집 가능한 페르소나로 복원 |
| 루미잉크 캐릭터 HTML | 편집 가능한 캐릭터 메모로 복원 |
| 루미잉크 로그 HTML | 원문과 설정이 있는 로그로 복원 |
| 그 밖의 HTML | 위험 요소를 정리한 자유 메모로 가져오기 |

프로젝트 HTML은 대상 프로젝트를 묻지 않고 새 프로젝트로 추가한다. 그 밖의 메모·World Info 파일은 저장할 프로젝트를 먼저 선택한다. 가져온 데이터의 이미지, 텍스트 길이, ID, 메모 유형 등은 허용 범위로 정규화된다.

전체 백업 HTML은 일반 `열기`가 아니라 `설정 → 백업 복원`에서 가져와야 한다.

설치형 PWA와 브라우저가 파일 핸들러 기능을 지원하는 환경에서는 운영체제의 `연결 프로그램` 또는 `다른 앱으로 열기`에서 루미잉크로 HTML 파일을 전달할 수도 있다. 전달받은 파일은 사이드바의 `파일 열기`와 같은 형식 판별 절차를 거친다.

## 15. 설정

### 15.1 밝게·어둡게

`테마`에서 라이트 모드와 다크 모드를 전환한다. 사이드바의 해·달 버튼으로도 즉시 바꿀 수 있다.

### 15.2 컬러 테마

앱 전체의 강조색, 버튼, 로고, 일부 배경과 로그 유형 표시색을 바꾼다. 라이트·다크 모드와 조합된다.

제공되는 20종은 다음과 같다.

`블루`, `핑크`, `그린`, `퍼플`, `골드`, `파스텔 블루`, `파스텔 핑크`, `파스텔 올리브`, `파스텔 퍼플`, `파스텔 골드`, `네이비`, `버건디`, `딥그린`, `딥바이올렛`, `럭셔리골드`, `무채색`, `브라운`, `메탈릭 실버`, `메탈릭 골드`, `블랙&화이트`

`테마와 연동` 프레임과 로그 메모 유형의 구분 표시는 이 컬러 기조를 따라간다.

### 15.3 웹폰트

기본 글꼴 외에 다음 프리셋을 제공한다.

- 프리텐다드
- 나눔명조
- 고운돋움
- IBM Plex Sans KR
- 본명조·Noto Serif KR

폰트 CSS 주소와 `font-family` 이름을 직접 입력해 다른 웹폰트도 적용할 수 있다. 사용자 지정 URL은 외부 서버에 폰트 파일을 요청하므로 신뢰할 수 있는 HTTPS 주소를 사용한다. `기본으로`를 누르면 시스템 기본 글꼴로 돌아간다.

### 15.4 앱 전체 글자 크기

- 작게: 0.9배
- 보통: 1배
- 크게: 1.12배

이 설정은 편집 문서 안의 개별 글자 크기가 아니라 앱 UI 전체의 확대 비율이다. 모바일 팝업도 이 확대 비율을 고려해 화면 높이에 맞춘다.

### 15.5 자유 메모 툴바 기본 상태

- 항상 표시
- 접어두기

접어두기를 선택해도 편집 중 왼쪽 펼치기 버튼으로 임시로 열 수 있다.

### 15.6 설치 아이콘

10종의 설치 아이콘 중 하나가 연결된 설치 페이지로 이동한다. 자세한 내용은 3.5절을 참고한다.

### 15.7 상세 매뉴얼

`설정 → 상세 매뉴얼`에서 앱에 포함된 전체 기능 설명서를 연다. 상단의 닫기 또는 `앱으로 돌아가기` 버튼을 누르면 앱 화면으로 돌아간다. 앱에서 매뉴얼을 연 경우 모바일 기기의 뒤로가기 버튼도 같은 방식으로 동작한다.

## 16. 데이터 저장과 백업

### 16.1 어디에 저장되는가

작업 데이터는 현재 사이트의 IndexedDB에 저장된다.

- `projects`: 프로젝트, 태그, 썸네일, 프레임, 고정 상태
- `notes`: 다섯 유형의 메모와 본문, 본문 이미지, 카드 이미지
- `files`: 자유 메모 첨부파일 Blob
- `backups`: 최근 자동 백업 스냅샷

다음과 같은 환경 설정은 localStorage에 저장된다.

- 밝게·어둡게
- 컬러 테마
- 앱 글자 크기
- 웹폰트 설정
- 정렬 방식
- 자유 메모 툴바 상태
- 최근 사용 글자색과 저장한 색
- 형광펜 색
- 사용자 로그 템플릿과 즐겨찾기
- 저장되지 않은 임시 초안

### 16.2 전체 백업

`설정 → 전체 백업`은 다음 내용을 하나의 `lumink-backup-날짜-시각.html` 파일로 저장한다.

- 모든 프로젝트
- 모든 메모
- 프로젝트 장식과 메모 설정
- 본문에 포함된 이미지
- 자유 메모 첨부파일

백업 HTML을 브라우저에서 열면 프로젝트와 메모 목록을 사람이 읽을 수 있는 요약이 보이고, 실제 복원 데이터는 문서 안의 JSON으로 포함된다.

전체 백업에는 앱 표시 설정, 사용자 로그 템플릿 목록, 템플릿 즐겨찾기가 별도 항목으로 포함되지 않는다. 다만 사용자 템플릿을 적용한 로그 메모는 템플릿 스냅샷을 메모 안에 보관하므로 그 메모의 디자인은 백업에 포함된다.

### 16.3 백업 복원 방식

`설정 → 백업 복원`에서 전체 백업 HTML을 선택하면 두 방식을 고른다.

#### 현재 데이터에 병합

- 현재 프로젝트와 메모를 유지한다.
- 백업과 현재 데이터의 ID가 같으면 백업 내용으로 덮어쓴다.
- 백업에만 있는 항목을 추가한다.
- 기본 메모함 중복은 기존 기본 메모함에 맞춰 정리한다.

#### 백업 시점으로 완전히 교체

- 현재 프로젝트, 메모, 첨부파일을 지운다.
- 선택한 백업 내용만 남긴다.
- 설정과 사용자 템플릿처럼 localStorage에 있는 값은 이 교체 대상이 아니다.

두 방식 모두 실제 복원 직전에 현재 작업 상태를 자동 백업으로 한 번 더 저장한다. 프로젝트·메모·첨부파일 쓰기는 하나의 IndexedDB 트랜잭션으로 처리해 일부만 교체되는 상황을 줄인다.

### 16.4 자동 백업

메모 저장을 계기로 자동 백업을 예약한다. 저장이 매우 잦을 때에는 최소 약 18초 간격으로 묶어 실행하며 최근 10개만 유지한다.

자동 백업에는 프로젝트, 메모, 첨부파일이 들어간다. `설정 → 자동 백업`에서 각 스냅샷의 날짜, 프로젝트 수, 메모 수, 첨부 수를 확인하고 병합 또는 완전 교체 방식으로 복원할 수 있다.

자동 백업은 같은 브라우저 저장공간 안에 있으므로 브라우저 사이트 데이터 자체를 삭제하면 함께 사라진다. 기기 고장이나 브라우저 초기화에 대비하려면 외부 파일로 저장하는 전체 백업이 필요하다.

### 16.5 저장되지 않은 임시 초안

자유 메모, 로어북, 로그, 페르소나, 캐릭터 메모의 텍스트 편집 중에는 localStorage에 별도의 임시 초안을 만든다.

- 메모별로 분리 저장
- 최대 보관 기간 14일
- JSON 문자열 기준 약 70만 자를 넘는 초안은 저장하지 않음
- 대용량 이미지 전체가 아닌 텍스트 중심 안전망
- 마지막 정상 저장보다 새롭고 실제 내용이 다를 때만 복구 여부 질문
- 사용자가 `복구` 또는 `버리기` 선택

이 기능은 비정상 종료나 갑작스러운 새로고침을 보완하지만 정식 백업을 대체하지 않는다.

### 16.6 저장공간 확인

설정의 `저장공간`은 브라우저가 제공하는 전체 사이트 사용량과 할당량, 자유 메모 첨부파일 수와 용량, 자동 백업 개수를 표시한다. 브라우저가 전체 사용량 API를 제공하지 않으면 첨부파일과 백업 정보만 표시한다.

이미지가 많으면 메모 데이터뿐 아니라 자동 백업도 커지므로 실제 사용량이 빠르게 증가할 수 있다.

### 16.7 데이터 초기화

데이터 초기화는 두 번의 확인을 거친 뒤 프로젝트, 메모, 첨부파일, 자동 백업을 모두 삭제한다. 이 작업은 되돌릴 수 없다.

현재 구현에서 테마·폰트 같은 표시 설정과 사용자 로그 템플릿 등 localStorage 항목은 작업 데이터 초기화 대상에 포함되지 않는다. 완전한 사이트 초기화가 필요하면 전체 백업 후 브라우저의 해당 사이트 데이터 삭제 기능을 사용해야 한다.

## 17. 삭제 복구와 데이터 정리 기능

### 17.1 6초 되돌리기

다음 삭제는 직후 약 6초 동안 되돌릴 수 있다.

- 프로젝트 삭제
- 메모 삭제
- 여러 프로젝트·메모 일괄 삭제
- 페르소나 갤러리 이미지 삭제
- 캐릭터 갤러리 이미지 삭제
- 캐릭터 페이지 삭제

새로운 되돌리기 작업이 시작되면 앞선 되돌리기 대기 상태가 교체될 수 있으므로, 실수했다면 바로 누르는 것이 좋다.

### 17.2 시작 시 데이터 보정

앱은 시작할 때 일부 오래된 데이터 구조를 현재 형식으로 보정한다.

- 중복 기본 프로젝트 통합
- 오래된 프레임 색상 키를 실제 색상 값으로 변환
- 잘못된 프레임 ID 제거
- 프로젝트를 잃은 메모를 기본 메모함으로 이동
- 오래된 자유 메모 구조를 현재 데이터 구조로 변환
- 14일이 지난 임시 초안 정리

## 18. 개인정보와 보안상 주의

### 18.1 로컬 우선

프로젝트와 메모 본문을 루미잉크 자체 서버로 전송하는 동기화 기능은 없다. 자료는 현재 브라우저 저장소에 남는다.

다만 웹폰트 프리셋이나 사용자가 지정한 외부 폰트 CSS를 불러올 때에는 해당 폰트 제공 서버에 일반적인 웹 요청이 발생할 수 있다. 이 요청에 메모 본문을 담아 보내지는 않는다.

### 18.2 가져오기 정화

외부 HTML을 자유 메모로 가져올 때에는 허용된 태그와 스타일만 남긴다. 스크립트, iframe, 폼, 임베드, 위험한 URL과 스타일은 제거한다. 외부 원격 이미지는 가져오지 않고 루미잉크가 생성하는 안전한 base64 이미지 형식만 수용한다.

로그 템플릿은 임의 HTML과 JavaScript를 허용하지 않고, 스타일 속성 허용 목록과 정규식 복잡도 검사를 사용한다.

### 18.3 직접 HTML 편집 주의

자유 메모와 크리에이터 메모의 HTML 소스 편집은 사용자가 직접 작성한 신뢰할 수 있는 내용에 사용한다. 인터넷에서 가져온 불명확한 코드를 그대로 붙여넣지 않는다.

### 18.4 브라우저 데이터 삭제 주의

다음 작업은 데이터 손실로 이어질 수 있다.

- 브라우저의 쿠키·사이트 데이터 전체 삭제
- 루미잉크 사이트 저장공간 삭제
- 시크릿 모드에서 작성 후 창 닫기
- 다른 도메인이나 다른 브라우저에서 기존 데이터가 자동으로 보일 것이라 기대하기
- 운영체제 또는 브라우저 프로필 초기화

중요한 자료는 전체 백업 파일을 브라우저 밖의 안전한 위치에 보관한다.

## 21. 권장 사용 순서

처음 사용하는 경우 다음 순서가 가장 이해하기 쉽다.

1. 작품 또는 세션 이름으로 프로젝트를 만든다.
2. 프로젝트 태그, 색상, 썸네일을 지정한다.
3. 세계관은 로어북, 플레이어 설정은 페르소나, 여러 NPC는 캐릭터 메모에 기록한다.
4. 자유 메모에는 플롯·아이디어·자료 파일을 보관한다.
5. 대화 결과는 로그 저장에 원문 그대로 넣고 템플릿과 이름 가림을 적용한다.
6. 프로젝트와 메모를 고정해 자주 쓰는 항목을 위에 둔다.
7. 작업 중에는 자동 저장을 사용하되 큰 수정 뒤에는 `Ctrl/⌘+S`로 즉시 저장한다.
8. 작업이 끝난 날에는 전체 백업 HTML을 외부 폴더나 클라우드 드라이브에 저장한다.

## 22. 현재 구현의 주요 입력 한도

| 항목 | 한도 또는 처리 방식 |
|---|---|
| 프로젝트 이름 | 최대 60자 |
| 프로젝트 태그 입력란 | 최대 500자 |
| 메모 이름 변경 입력란 | 최대 80자 |
| 로그 원문 | 저장 정규화 기준 최대 500,000자 |
| 로어북 원문 | 복원·가져오기 정규화 기준 최대 500,000자 |
| 로그 가림 원본 이름 | 최대 20개, 각 최대 80자 |
| 로그 대체 표시 | 최대 80자 |
| 사용자 로그 템플릿 | 최대 20개, 파일당 200KB 이하 |
| 템플릿 정규식 규칙 | 템플릿당 최대 20개 |
| 본문·카드 이미지 입력 | 파일당 12MB 이하, 2,400만 픽셀 이하 |
| 자유 메모 일반 첨부 | 파일당 15MB 이하 |
| 임시 초안 | JSON 문자열 약 700,000자 이하, 최대 14일 |
| 자동 백업 | 최근 10개 |

표에 없는 본문·갤러리·캐릭터 페이지의 실질적인 최대치는 브라우저가 해당 사이트에 허용한 저장공간의 영향을 받는다. 대량 이미지를 넣기 전에는 설정의 저장공간 표시를 확인하고 전체 백업을 만들어 두는 것이 안전하다.

## 부록 B. 가장 중요한 백업 원칙

1. 자동 저장은 편집 실수를 줄이지만 기기 고장을 막아 주지는 않는다.
2. 임시 초안은 비정상 종료를 보완하지만 14일이 지나면 정리된다.
3. 자동 백업은 같은 브라우저 안에 있으므로 사이트 데이터 삭제와 함께 사라질 수 있다.
4. 다른 기기로 옮길 때에는 반드시 `설정 → 전체 백업` 파일을 사용한다.
5. 자유 메모 하나의 HTML에는 별도 첨부파일이 들어가지 않는다.
6. 사용자 로그 템플릿 목록과 즐겨찾기를 별도로 보관하려면 원본 JSON 템플릿 파일도 함께 보관한다.

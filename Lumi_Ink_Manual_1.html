v1.7 현재 기준 빠른 안내

이 문서는 v1.7 기준의 루미잉크 사용 흐름을 반영합니다. 정규식 작업실, 자유 메모 삽입 도구, 아이디어 보드 확장, 퀵메뉴 슬롯 조정, 자동 백업 슬롯 관리가 포함됩니다.

저장 상태 표시

각 메모 상단의 기록 중, 저장됨 텍스트 왼쪽에는 상태 점이 표시됩니다. 기록 중에는 현재 강조색, 저장됨에는 초록색 점이 켜집니다. 이 표시는 앱 내부 저장 상태를 뜻하며, 전체 백업 파일을 만든 상태를 뜻하지는 않습니다.

자유 메모

첨부 오른쪽 삽입 메뉴에서 내 글 바로가기와 구분선을 넣습니다. 내 글 바로가기는 아이디어 보드형 카드로 표시되며 버튼 컬러와 글자색을 선택할 수 있습니다.

정규식 작업실

IN/OUT/sample을 편집하고 실시간 검증, 캡처 미리보기, 샌드박스 미리보기, SillyTavern 업로드용 JSON 내보내기를 사용합니다. sample만 있어도 IN 자동생성을 시작할 수 있습니다.

퀵메뉴

설정에서 슬롯 수를 1~10개로 조정합니다. 신규 설치 기본값은 5개이며, 슬롯 수 설정이 없던 기존 사용자는 기존 7개 슬롯을 보존합니다.

아이디어 보드

메모지, 이미지, GIF, 음악, 영상, 파일, 내 메모 링크를 배치하고 색상, 글자색, 프레임, 그룹, 스냅, HTML/PNG 내보내기를 조합합니다.

01

루미잉크란 무엇인가

루미잉크는 창작 자료를 프로젝트 단위로 정리하는 로컬 우선 메모 웹앱입니다. 리치 문서형 자유 메모, 원본 코드 보존용 코드 작업실, AI 롤플레이용 로어북·로그, 페르소나·캐릭터를 단일/다인으로 구성하는 인물 카드, 그리고 자유 배치형 아이디어 보드를 함께 제공합니다.

앱의 기본 구조

  1. 가장 바깥에 프로젝트가 있습니다.
  2. 프로젝트 안에서 문서·코드·세계관·인물·아이디어를 목적에 맞는 메모 타입으로 관리합니다.
  3. 인물 카드는 페르소나 / 다인 페르소나와 캐릭터 / 다인 캐릭터를 각각 같은 카드 구조 안에서 단일·다인으로 전환하며 관리합니다.
  4. 아이디어 보드는 위치·크기·회전·색상·첨부 참조까지 보존하는 독립 캔버스입니다.
  5. 작성 데이터는 계정 서버가 아니라 현재 브라우저의 로컬 저장공간에 보관됩니다.
  6. 다른 기기나 브라우저로 옮길 때는 전체 백업 파일을 사용합니다.

로그인·클라우드 동기화가 없습니다. 같은 기기라도 다른 브라우저, 다른 브라우저 프로필, 시크릿 모드, 다른 배포 주소는 서로 다른 저장공간으로 취급될 수 있습니다.

따라서 중요한 자료는 주기적으로 전체 백업해야 합니다.

03

설치와 실행

3.1 웹앱으로 사용하기

배포된 루미잉크 주소를 브라우저에서 열면 별도 설치 없이 사용할 수 있습니다. 다만 브라우저 데이터 삭제, 사이트 데이터 삭제, 시크릿 모드 종료 등으로 로컬 데이터가 사라질 수 있으므로 장기 사용 전에는 설치형 PWA로 추가하고 백업 습관을 잡는 편이 좋습니다.

3.2 Android

  1. Chrome에서 루미잉크 주소를 엽니다.
  2. 브라우저의 점 세 개 메뉴를 엽니다.
  3. 앱 설치 또는 홈 화면에 추가를 선택합니다.
  4. 설치된 루미잉크 아이콘으로 실행합니다.

3.3 iPhone·iPad

  1. Safari에서 루미잉크 주소를 엽니다.
  2. 공유 버튼을 누릅니다.
  3. 홈 화면에 추가를 선택합니다.

3.4 데스크톱

Chrome 또는 Edge의 주소창 설치 아이콘을 누르거나, 브라우저 메뉴에서 앱 설치를 선택합니다. 설치 후에는 독립 창 형태로 실행할 수 있습니다.

3.5 설치 아이콘 선택

설정 → 앱 설치 아이콘에서 다음 10종을 고를 수 있습니다.

잉크 블루골드바이올렛로즈포레스트파스텔 블루파스텔 핑크파스텔 그린파스텔 퍼플파스텔 옐로우

아이콘을 선택하면 해당 아이콘이 연결된 설치 전용 페이지로 이동합니다. 이미 설치한 PWA의 아이콘은 선택만으로 즉시 교체되지 않을 수 있습니다. 이 경우 기존 설치본을 제거하고 원하는 아이콘 페이지에서 다시 설치해야 합니다.

3.6 오프라인 사용과 업데이트

핵심 앱 파일, 토크나이저, 아이콘, 로그 템플릿 등은 서비스 워커 캐시에 저장됩니다. 한 번 정상적으로 불러온 뒤에는 네트워크가 끊겨도 대부분의 작성·열람 기능을 사용할 수 있습니다.

앱의 HTML과 JavaScript는 온라인일 때 새 파일을 먼저 확인하고, 네트워크를 사용할 수 없을 때 캐시로 돌아갑니다. 업데이트가 배포되었는데 이전 화면이 남아 있다면 온라인 상태에서 앱을 다시 열거나 새로고침합니다.

웹폰트를 외부 주소로 설정한 경우 해당 폰트의 최초 로드는 인터넷 연결이 필요할 수 있습니다. 폰트를 불러오지 못해도 작성 데이터가 사라지지는 않으며 대체 글꼴로 표시됩니다.

04

화면과 기본 조작

4.1 홈 화면

홈 화면은 프로젝트와 최근 메모를 빠르게 여는 시작 화면입니다.

메뉴 버튼
왼쪽 상단 — 사이드바를 엽니다.
로고
가운데 Lumi Ink 로고 — 홈 목록을 맨 위로 부드럽게 이동합니다.
설정 버튼
오른쪽 상단 — 설정 화면을 엽니다.
검색창
프로젝트와 메모를 통합 검색합니다.
새 메모
메모 유형과 저장할 프로젝트를 순서대로 선택합니다.
새 프로젝트
이름과 태그를 입력해 프로젝트를 만듭니다.
프로젝트 영역
프로젝트 카드, 메모 수, 수정 시각을 표시합니다.
최근 수정 메모
최대 10개의 최근 메모를 표시합니다.

프로젝트 수가 적어 프로젝트 영역 자체에 스크롤이 생기지 않더라도, 그 영역에서 위아래로 휠이나 스와이프하면 홈 전체 화면의 스크롤로 자연스럽게 이어집니다. 프로젝트 목록이 긴 경우에는 목록의 위·아래 경계에서 바깥 홈 스크롤로 연결됩니다.

4.2 사이드바

어느 주요 화면에서든 왼쪽 위 메뉴 버튼을 누르면 사이드바를 열 수 있습니다. 모바일에서는 화면 왼쪽 가장자리에서 오른쪽으로 스와이프해도 열립니다.

사이드바에는 다음 항목이 있습니다.

  • 전체 프로젝트와 각 프로젝트의 메모 수
  • 최근 수정한 메모 최대 10개
  • 새 메모 · 새 프로젝트
  • 열기 — HTML 또는 JSON 파일 가져오기
  • 설정
  • 밝게/어둡게 빠른 전환

사이드바의 Lumi Ink 로고를 누르면 홈으로 돌아갑니다.

4.3 뒤로 가기와 겹친 창 닫기

브라우저 또는 기기의 뒤로 가기는 먼저 현재 화면 위에 열린 요소를 닫습니다. 선택 모드 → 이미지 자르기 → 이미지 확대 화면 → 팝업 → 하단 메뉴 → 사이드바 순으로 가장 위의 요소를 정리한 뒤 이전 화면으로 이동합니다.

작성 화면에서 뒤로 가거나 다른 메모를 열 때에는 현재 편집 내용을 먼저 저장한 뒤 화면을 전환합니다.

4.4 길게 누르기와 우클릭

프로젝트 카드나 메모 카드를 길게 누르면 해당 항목의 관리 메뉴가 열립니다. 데스크톱에서는 우클릭으로 같은 메뉴를 열 수 있습니다.

로그 템플릿은 길게 누르기의 의미가 다릅니다. 템플릿 선택 창에서 길게 누르면 템플릿이 즐겨찾기에 추가되거나 해제됩니다.

4.5 저장 상태 표시

각 편집 화면의 상단에는 기록 중, 저장됨 상태가 잠시 표시됩니다. 일반적인 텍스트 입력은 약 0.55초 동안 추가 입력이 없으면 자동 저장됩니다. 화면을 떠날 때에도 남아 있는 편집 내용을 저장합니다.

상단 저장 버튼을 누르거나 Windows/Linux에서는 Ctrl+S, macOS에서는 ⌘+S를 누르면 즉시 앱 내부에 저장할 수 있습니다. 이 단축키는 자유 메모, 코드 작업실, 로어북, 로그, 통합 인물 카드, 아이디어 보드 편집 화면에서 동작합니다. 코드 작업실에서 파일로 내려받으려면 상단 작업 막대의 파일 저장 또는 Ctrl/⌘+Shift+S를 사용합니다.

05

프로젝트 관리

5.1 프로젝트 만들기

처음 실행하면 삭제할 수 없는 기본 메모함이 자동으로 준비됩니다. 프로젝트를 따로 나누고 싶지 않은 자료나 가져오기 과정에서 위치를 잃은 메모를 안전하게 모으는 기본 위치입니다.

  1. 홈이나 사이드바에서 새 프로젝트를 누릅니다.
  2. 프로젝트 이름을 입력합니다. 이름은 필수이며 최대 60자입니다.
  3. 선택적으로 태그를 쉼표로 구분해 입력합니다. 입력란 전체는 최대 500자입니다.
  4. 만들기를 누릅니다.

프로젝트 편집 창에는 과거 호환을 위해 내부적으로 설명 필드가 남아 있지만, 현재 화면에서는 쉼표로 나눈 값을 프로젝트 태그 칩으로 표시합니다. 예를 들어 판타지, 로맨스, 진행중을 입력하면 프로젝트 상세 상단에 세 개의 태그가 나타납니다.

5.2 프로젝트 상세 화면

프로젝트를 열면 다음 정보가 보입니다.

  • 프로젝트 썸네일
  • 프로젝트 이름
  • 전체 메모 수
  • 프로젝트 태그
  • 유형별 메모 목록

메모 섹션은 페르소나 → 캐릭터 → 로그 저장 → 로어북 → 코드 작업실 → 아이디어 보드 → 자유 메모 순서로 표시됩니다. 각 섹션 안에서는 먼저 고정된 메모가 나오고, 나머지 메모가 선택한 정렬 방식으로 배치됩니다.

5.3 프로젝트 정렬

홈의 정렬 버튼에서 다음 네 가지를 선택할 수 있습니다.

최신순오래된순이름순 ㄱ→ㅎ이름순 ㅎ→ㄱ

프로젝트 상세 화면의 첫 메모 섹션에 있는 정렬 버튼은 모든 메모 섹션의 정렬 기준을 함께 바꿉니다. 홈 프로젝트 정렬과 프로젝트 안 메모 정렬은 서로 별도로 저장됩니다.

5.4 프로젝트 관리 메뉴

프로젝트 카드의 점 세 개 버튼을 누르거나 카드를 길게 누르면 다음 작업을 할 수 있습니다.

  • 새 메모 만들기
  • 상단 고정 또는 고정 해제
  • 이름·태그 편집
  • 프로젝트 색상 지정
  • 프로젝트 파일로 내보내기
  • 프로젝트 전체 복제
  • 프로젝트 아이콘 변경
  • 썸네일 프레임 선택
  • 소속 로어북을 World Info로 묶어 내보내기
  • 프로젝트 삭제

홈 화면에서는 선택 항목도 표시되어 여러 프로젝트를 한꺼번에 삭제할 수 있습니다.

5.5 프로젝트 고정

프로젝트는 최대 4개까지 상단에 고정할 수 있습니다. 고정된 프로젝트끼리는 고정한 순서를 기준으로 표시되고, 아래에 나머지 프로젝트가 이어집니다.

5.6 프로젝트 색상

프로젝트 카드의 경계와 상세 썸네일 강조에 사용할 색상을 지정할 수 있습니다. 제공되는 색상은 다음 10종입니다.

루미 핑크코랄 잉크선셋 오렌지골드 글로우포레스트 그린아쿠아 민트네온 시안미드나잇 블루라벤더 퍼플미스티 그레이

색상 없음으로 되돌리는 항목도 있습니다.

5.7 프로젝트 썸네일

기본 제공 썸네일 중 하나를 고르거나 사용자 이미지를 직접 업로드할 수 있습니다.

별 고래분홍 창가황금 옥좌얼음 성빛의 숲길야경 도시푸른 하늘성분홍 공주방깃펜과 편지수정 첨탑보라 수정달빛 서재이상한 나라겨울 보름달라벤더 정원산호 바다

업로드 이미지는 정사각형 자르기 화면에서 위치와 확대 비율을 조절한 뒤 512×512 이미지로 저장합니다.

5.8 썸네일 프레임

프로젝트 썸네일 바깥에는 20종의 장식 프레임을 적용할 수 있습니다.

클래식비드바로크아르데코로프고딕보태니컬스캘럽스타라이트로열슬림 헤어라인슬림 코너 브래킷슬림 인셋 닷슬림 파인 비드슬림 다이아 코너슬림 미니 노치슬림 덩굴슬림 물결슬림 점선슬림 스파클

프레임을 고른 뒤 고정 색상, 테마와 연동, 투명 글래스, 글래시한 명암 중 하나를 선택합니다. 팔레트 마지막의 직접 색상 선택에서는 정사각형 색상판·색조 슬라이더·HEX·RGB 입력으로 원하는 프레임색을 만들고 저장할 수 있습니다.

테마와 연동
앱의 컬러 테마를 바꾸면 프레임 색도 함께 바뀝니다.
투명 글래스
반투명 흰색 유리 프레임처럼 표시됩니다.
글래시한 명암
반투명 하이라이트와 그림자를 사용해 깊이를 표현합니다.

프레임은 홈 카드, 사이드바, 프로젝트 선택 창, 프로젝트 상세 헤더에서 같은 구성으로 표시됩니다.

5.9 프로젝트 복제와 삭제

프로젝트 복제는 프로젝트 정보뿐 아니라 내부의 모든 메모와 파일 참조를 새 ID로 복사합니다. 아이디어 보드의 이미지·음악·영상·첨부·캔버스 배경도 복제본 전용 파일 레코드로 다시 연결됩니다. 복제본 이름에는 (사본)이 붙습니다.

프로젝트를 삭제하면 소속 메모와 그 첨부파일도 함께 삭제됩니다. 삭제 직후 약 6초 동안 표시되는 되돌리기를 누르면 프로젝트, 메모, 첨부파일을 복원할 수 있습니다. 삭제 전에는 자동 백업도 한 번 실행합니다.

5.10 프로젝트 단위 내보내기와 불러오기

프로젝트 관리 메뉴의 프로젝트 파일로 내보내기는 프로젝트 정보, 소속 메모 전체, 본문 이미지·첨부파일·아이디어 보드 파일·캔버스 배경을 하나의 HTML 패키지로 저장합니다. 사용자 템플릿을 적용한 로그는 저장된 템플릿 스냅샷과 이름 가림 설정도 함께 포함합니다.

다른 루미잉크에서 사이드바의 열기로 이 HTML을 선택하면 기존 데이터를 건드리지 않고 새 프로젝트로 가져옵니다. 프로젝트·메모·첨부파일에는 모두 새 ID가 발급되며, 같은 이름이 있으면 (가져옴), (가져옴 2)처럼 자동으로 구분됩니다.

프로젝트 HTML은 한 프로젝트를 공유하거나 옮기는 용도입니다. 앱 전체를 보존하거나 기기를 이전할 때는 설정 → 전체 백업을 사용합니다.

06

메모 만들기와 공통 관리

6.1 새 메모 만들기

  1. 홈이나 사이드바에서 새 메모를 누릅니다.
  2. 상단에는 생성할 메모 타입을 골라주세요 안내와 자유 메모 빠른 생성 카드가 표시됩니다. 빠른 생성 카드를 누르면 자유 메모 생성 흐름으로 바로 들어갑니다. 이 카드는 기존의 블루 그라데이션과 흰색 아이콘·텍스트 구성을 유지하며, 카드 아래 그림자는 짧고 더 흐리게만 표시됩니다.
  3. 유형을 고를 때는 캐릭터 · 작업실 · 아뜰리에 탭을 사용합니다. 캐릭터 탭에는 페르소나 · 다인 페르소나 · 캐릭터 · 다인 캐릭터, 작업실 탭에는 로어북 · 정규식 작업실 · 코드 작업실, 아뜰리에 탭에는 자유 메모 · 로그 저장 · 아이디어 보드가 있습니다.
  4. 저장할 프로젝트를 선택합니다.
  5. 각 타입의 제목·내용을 작성하면 자동 저장됩니다.

6.2 자동 제목과 수동 제목

메모 유형자동 제목 규칙
자유 메모본문의 첫 번째 비어 있지 않은 줄을 최대 52자로 사용
로어북기본값 이름 없는 로어북; 필요할 때 직접 변경
로그원문 첫 줄에서 따옴표·대괄호·별표 등의 표식 일부를 제거해 최대 52자로 사용
페르소나한국어 이름, 없으면 영어 이름 사용
캐릭터 메모통합 인물 카드의 첫 페이지 이름을 사용하고 캐릭터 모음이면 이름 외 N명 형식으로 사용

메모 관리 메뉴에서 이름을 직접 바꾸면 해당 제목은 잠금 상태가 되어 이후 본문이나 인물 이름이 바뀌어도 자동 제목으로 덮어쓰지 않습니다.

6.3 메모별 색상

각 메모에는 프로젝트 색상과 별개로 색상 표식을 지정할 수 있습니다. 아이디어 보드 안의 메모지·음악·메모 인용·첨부파일·캔버스 오버레이는 이 표식과 별도로 앱의 20종 테마 컬러를 개별 선택할 수 있습니다.

6.4 메모 고정

프로젝트 상세의 각 메모 유형 섹션마다 최대 3개를 고정할 수 있습니다. 예를 들어 자유 메모 3개와 로그 3개를 각각 고정할 수 있습니다.

홈의 최근 메모와 사이드바의 최근 메모에는 서로 독립적인 고정 상태가 있습니다. 각 위치에서 최대 3개까지 상단에 고정할 수 있습니다. 홈 고정과 사이드바 고정은 프로젝트 상세의 메모 고정과 별개입니다.

6.5 메모 관리 메뉴의 공통 작업

메모 유형에 따라 세부 항목은 다르지만 일반적으로 다음 기능을 제공합니다.

  • 선택 모드 진입
  • 상단 고정 또는 해제
  • 이름 바꾸기
  • 색상 지정
  • 해당 유형의 파일로 내보내기
  • 다른 프로젝트로 이동
  • 선택한 프로젝트에 복제
  • 삭제

복제할 때 자유 메모 첨부파일도 새 파일 레코드로 함께 복사됩니다. 삭제 직후에는 약 6초 동안 되돌릴 수 있습니다.

07

자유 메모

7.1 용도와 화면 구분

자유 메모는 일반 문서를 쓰는 리치 에디터입니다. 처음 열면 읽기 화면이 나타나며, 상단 편집 버튼이나 본문 더블클릭으로 편집 화면에 들어갑니다. 문서 안에 넣은 사진과 동영상은 저장된 HTML의 인라인 크기값으로 관리됩니다.

툴바의 미디어 버튼을 누르면 사진 또는 동영상을 파일선택기로 불러올 수 있습니다. 동영상은 아이디어 보드와 같은 파일당 30MB 이하 제한을 적용하며, 삽입 뒤 사진과 동일한 크기 조절 핸들·가로세로 입력·비율 고정을 사용할 수 있습니다.

읽기 화면
완성된 본문과 첨부파일을 열람합니다.
편집 화면
리치 에디터 또는 HTML 소스로 내용을 수정합니다.

읽기 화면의 본문을 길게 누르면 서식을 제외한 본문 전체 텍스트를 클립보드에 복사합니다.

7.2 서식 도구

편집 툴바에서 다음 기능을 사용할 수 있습니다.

굵게기울임밑줄취소선글자 크기 줄이기·늘리기글자 크기 목록 선택글자 색형광펜왼쪽·가운데·오른쪽 정렬글머리표 목록번호 목록이미지 삽입일반 파일 첨부하이퍼링크코드 블록서식·코드·링크 지우기실행 취소HTML 소스 보기·편집툴바 접기·펼치기

7.3 글자 크기

텍스트를 선택한 뒤 크기 목록에서 다음 일곱 단계 중 하나를 고를 수 있습니다.

아주 작게 12px작게 14px보통 16px크게 19px더 크게 23px제목 28px대제목 34px

A−, A+는 현재 크기를 한 단계씩 바꿉니다.

7.4 글자 색

기본 팔레트, 운영체제 색상 선택기, HEX 직접 입력을 지원합니다. 마음에 드는 색을 내 색상에 저장할 수 있으며 최대 24개까지 유지합니다. 마지막으로 적용한 색도 다음 편집 때 기억합니다.

7.5 형광펜

텍스트를 선택한 뒤 형광펜 버튼을 짧게 누르면 현재 형광펜 색을 적용하거나 이미 적용된 형광펜을 제거합니다. 버튼을 약 0.48초 동안 길게 누르면 다음 색상 선택 창이 열립니다.

하늘색핑크색연두색연보라색노란색

선택한 형광펜 색은 다음 사용에도 유지되며 캐릭터 크리에이터 메모의 형광펜과 공유됩니다.

7.6 HTML 소스 편집

<> 형태의 코드 버튼을 누르면 현재 메모의 HTML 소스를 직접 볼 수 있습니다. 소스 모드에서 수정한 HTML은 일반 편집 모드와 같은 본문 데이터로 저장됩니다.

이 기능은 고급 사용자용입니다. 앱 안에서 직접 입력한 소스는 가져온 외부 HTML과 달리 별도의 전체 정화 과정을 거치지 않으므로, 출처를 알 수 없는 HTML·이벤트 속성·스크립트성 코드를 붙여넣지 않는 것이 안전합니다.

7.7 이미지 삽입

이미지는 문서 본문 안에 삽입됩니다. 긴 변을 최대 1280px로 줄이고 JPEG 데이터로 변환한 뒤 메모 HTML 안에 포함합니다.

  • 파일 크기 12MB 이하
  • 전체 해상도 2,400만 픽셀 이하
  • 일반 이미지 파일만 허용
  • SVG는 업로드 이미지로 허용하지 않음

7.8 파일 첨부

본문 이미지와 별개로 일반 파일을 첨부할 수 있습니다.

  • 한 파일당 최대 15MB
  • 이미지, 문서, 압축 파일, 오디오, 비디오 등 파일 유형에 맞춘 아이콘 표시
  • 읽기·편집 화면에서 파일 다운로드 가능
  • 편집 화면에서 첨부 삭제 가능
  • 첨부 삭제는 확인 창을 거치지만 즉시 되돌리기 버튼은 제공하지 않음

첨부파일은 메모 본문이 아니라 브라우저의 별도 파일 저장소에 보관됩니다. 프로젝트·메모 복제와 전체 백업에는 포함되지만, 자유 메모 하나를 일반 HTML로 저장할 때에는 첨부파일이 포함되지 않습니다.

7.9 링크 · 내 글 바로가기 · 구분선

링크를 걸 텍스트를 먼저 선택한 뒤 링크 버튼을 누르고 주소를 입력합니다. https://를 생략하면 자동으로 붙입니다.

툴바의 첨부파일 오른쪽 ＋ 버튼에서는 내 글 바로가기와 구분선을 현재 커서 위치에 삽입합니다. 내 글 바로가기는 제목·프로젝트 검색으로 다른 메모를 고른 다음, 테마 팔레트 또는 직접 색상으로 버튼 컬러를 정합니다. 읽기 화면에서는 같은 메모를 열어 주며, 연결 대상이 프로젝트를 옮겨도 메모 ID를 따라가므로 링크가 유지됩니다. 구분선은 아이디어 보드와 같은 굵기 → 디자인 → 컬러 선택 흐름을 거쳐 삽입됩니다. 57종 디자인, 1~12px 굵기, 테마 팔레트와 직접 색상을 지원하며 선택한 값은 본문과 함께 저장됩니다. 삽입된 구분선은 편집기와 읽기 화면의 실제 본문 폭을 따라 유동적으로 표시됩니다.

  • http:// 또는 https:// 주소를 입력한 뒤 공백이나 Enter를 누르면 자동 링크 변환
  • 선택한 텍스트 위에 URL만 붙여넣으면 해당 텍스트에 링크 적용
  • 선택 영역 없이 URL만 붙여넣으면 URL 자체를 링크로 삽입
  • 읽기 화면의 링크는 새 창에서 열리고 noopener noreferrer 보호 속성을 사용

7.10 코드 블록

코드 블록 버튼은 현재 문단을 pre 형식으로 바꿉니다. 읽기 화면에서는 각 코드 블록 오른쪽 위에 복사 버튼이 자동으로 붙습니다.

지우개 버튼은 선택 영역의 일반 서식, 링크, 코드 블록·코드 요소를 함께 제거합니다.

7.11 자유 메모 HTML 저장과 다시 열기

메모 관리 메뉴의 .html로 저장은 본문 서식과 본문 안 이미지가 포함된 일반 HTML 문서를 만듭니다. 이 파일은 브라우저에서 독립적으로 열 수 있습니다.

  • 별도 첨부파일은 포함되지 않습니다.
  • 루미잉크 고유 메타데이터를 넣는 페르소나·캐릭터·로그 HTML과 달리, 자유 메모 HTML은 일반 문서 형식입니다.
  • 루미잉크의 열기로 다시 가져오면 HTML 제목과 정화된 본문을 사용해 새 자유 메모를 만듭니다.
  • 외부 HTML을 가져올 때 스크립트, 폼, iframe, 위험한 속성, 원격 이미지 등은 제거하거나 제한합니다.

7.12 에디터 이미지 크기 조절

편집 중 본문 이미지를 누르면 선택 테두리와 조절 패널이 나타납니다. 네 모서리 핸들을 드래그해 크기를 바꾸거나, 가로·세로 픽셀값을 직접 입력할 수 있습니다.

  • 비율 고정: 기본값입니다. 한 변을 바꾸면 원본 비율을 유지해 다른 변도 같이 계산합니다.
  • 비율 해제: 가로·세로를 독립적으로 입력하거나 조절합니다.
  • 자동 크기: 직접 지정한 폭·높이를 지우고 문서 레이아웃에 맞는 기본 크기로 돌립니다.

선택 테두리와 입력 패널은 저장되지 않습니다. 실제 문서에는 이미지의 width와 height 스타일만 남으므로, HTML 저장·백업·복원 뒤에도 조정한 크기가 유지됩니다.

7.13 HTML을 문서로 가져올 때

외부 HTML을 자유 메모로 열면 문서 안전성을 위해 구조·정적 스타일·정적 SVG를 폭넓게 허용하면서 위험 요소를 제거합니다. 인라인 레이아웃과 대부분의 정적 디자인은 유지되지만, 스크립트·폼·iframe·외부 스타일시트·원격 실행 요소는 보존 대상이 아닙니다. 코드 원본 자체를 편집·보관하려면 다음의 코드 작업실을 사용합니다.

07+

코드 작업실

코드 작업실은 상태창, 배너, 홍보글, 커스텀 카드와 HTML·JSON·Markdown 원문처럼 소스 또는 문법 원문 자체가 작업물인 파일을 위한 전용 메모입니다. 자유 메모와 달리 원본을 리치 에디터의 innerHTML로 다시 직렬화하지 않으며, 가져온 문자열을 그대로 보관합니다.

원본 보존과 화면 안전성은 분리됩니다. 저장하는 소스는 손대지 않지만, 미리보기는 별도 샌드박스 안에서 위험 요소를 제거한 사본으로 렌더링합니다.

HTML · JSON · Markdown 원문으로 가져오기

  1. 사이드바의 열기에서 HTML, JSON 또는 Markdown 파일을 선택합니다.
  2. 일반 JSON은 먼저 JSON 열기 창이 표시됩니다. 코드 작업실로 원본 열기를 선택하면 JSON 문자열을 그대로 편집할 수 있습니다.
  3. Markdown 파일(.md, .markdown 등)은 코드 작업실에서 바로 열립니다. SOURCE에서는 원문 그대로 편집하고, PREVIEW·SPLIT에서는 렌더링된 문서로 확인합니다.
  4. 파일명 확장자·MIME 정보가 누락되어도 내용이 { 또는 [로 시작하는 JSON이면 JSON 열기 흐름으로 인식합니다. 문법이 깨진 JSON도 원문 작업실에서 점검할 수 있습니다.
  5. 파일 내용은 정화되지 않은 원본 문자열로 저장됩니다. JSON 파일은 원본 작업실을 선택하면 .json 저장이 기본값으로 잡힙니다.

소스 · 미리보기 · 분할 보기

상단의 SOURCE, PREVIEW, SPLIT 버튼으로 화면을 전환합니다. 소스 화면에서는 원문을 직접 편집하고, 분할 보기에서는 원문과 확인 화면을 나란히 볼 수 있습니다.

HTML은 iframe sandbox 안에서 렌더링한 사본을 미리보기로 표시합니다. 스크립트, 이벤트 속성, 프레임, 폼, 자동 새로고침과 위험한 URL은 미리보기 사본에서 제거됩니다. JSON은 실행하지 않는 원문 보기로 표시하고, Markdown은 제목·목록·인용문·체크리스트·표·코드 블록·링크·이미지를 렌더링한 문서 보기로 표시합니다. Markdown 안의 HTML은 실행하지 않고 텍스트로 처리합니다. HTML 전용 웹페이지로 열기 버튼은 JSON·Markdown에서 비활성화됩니다.

저장·내보내기·제한

원본은 최대 5MiB까지 저장할 수 있습니다. 작업 막대의 파일 저장 또는 더보기 메뉴에서 파일 이름과 .html/.json/.md 확장자를 선택할 수 있습니다. .json은 저장 전 문법을 검사하며, 문법 오류가 있으면 파일을 만들지 않습니다. 내보내기는 앱 래퍼를 덧붙이지 않고 저장 문자열을 그대로 파일로 만듭니다.

07R

정규식 작업실

정규식 작업실은 SillyTavern의 Regex 스크립트를 앱 안에서 작성하고, 샘플 입력으로 먼저 검증한 뒤 바로 가져갈 수 있는 JSON으로 저장하는 전용 작업대입니다. 코드 작업실처럼 원문을 보관하는 공간이 아니라, findRegex·replaceString와 SillyTavern 옵션을 구조화해 다룹니다.

✦

호환 범위: 이 작업실은 findRegex와 replaceString을 가진 SillyTavern Regex JSON을 인식합니다. 기존 JSON을 사이드바 열기로 선택한 뒤 정규식 작업실로 가져오기를 고르면 같은 옵션을 편집할 수 있습니다.

작성 흐름

  1. scriptName에 스크립트 이름을 적습니다. 제목을 따로 고정하지 않았다면 이 이름이 메모 제목에도 반영됩니다.
  2. IN · findRegex에 찾을 정규식을 입력합니다. /패턴/플래그 리터럴과 일반 패턴 모두 입력할 수 있습니다.
  3. OUT · replaceString에 치환 결과를 넣습니다. 캡처 그룹은 $1, $2처럼 참조합니다.
  4. sample에 실제 대화에서 들어올 예시를 넣고, 유동 값은 [[값]]으로 감싸면 샘플→IN이 해당 부분을 캡처 그룹으로 만드는 패턴 초안을 생성합니다. 완성된 [[…]] 구간은 입력창 안에서만 컬러 토큰 박스로 보이므로, 실제 원문을 바꾸지 않고 유동 값을 빠르게 구분할 수 있습니다.
  5. PREVIEW 또는 SPLIT에서 매치 수, 첫 매치의 캡처 값, OUT HTML 렌더링 결과를 확인한 뒤 JSON으로 저장합니다.

정상, 검증 실패, 미리보기 대기, 단일 정상 결과의 매치, 여러 결과의 n개 매치 같은 상태 문구는 작업 흐름을 방해하지 않도록 보조 정보 크기로 표시됩니다. 오류가 나면 IN의 JavaScript 정규식 문법 오류 메시지가 함께 표시됩니다.

샘플 도구와 글로벌 플래그

  • 글로벌 /g: 켜면 IN을 /.../g 형태로 맞춥니다. 이미 리터럴로 입력한 경우 플래그만 조정하고, 일반 패턴은 리터럴 형태로 감쌉니다.
  • 샘플→IN: [[값]] 표식을 캡처 그룹으로 바꾸고, 공백·쉼표·구분기호에는 느슨한 공백 허용을 적용한 초안 패턴을 만듭니다.
  • 샘플 자동: IN이 문법상 유효하면 OUT이 비어 있어도 캡처 구조를 바탕으로 sample 초안을 생성합니다. OUT에 $n이 있으면 그 참조 범위도 함께 고려합니다.

로어북

8.1 용도

로어북은 SillyTavern World Info 항목을 작성하고 관리하는 메모 유형입니다. 원문은 경량 마크다운으로 저장하며, 발동 키워드와 constant, 삽입 깊이를 함께 기록합니다.

8.2 발동 키워드

상단 발동 키워드 입력란에 값을 입력한 뒤 Enter 또는 쉼표를 누르면 키워드 칩이 생성됩니다.

  • 한 번에 쉼표로 여러 키워드 추가 가능
  • 동일한 문자열의 중복 추가 방지
  • 각 칩의 × 버튼으로 개별 삭제
  • World Info 내보내기 때 key 배열로 전달

8.3 항상 활성

항상 활성 (constant)을 켜면 키워드 일치 여부와 관계없이 항상 활성화되는 World Info 항목으로 내보냅니다. JSON에서는 constant: true가 됩니다.

눈 모양 버튼으로 원문 편집과 렌더링된 미리보기를 전환합니다. 미리보기 화면을 길게 누르면 꾸며진 화면이 아니라 원본 마크다운 전체를 클립보드에 복사합니다.

8.7 토큰 계산

로어북 본문은 번들된 cl100k 계열 토크나이저로 토큰 수를 계산합니다. 토크나이저는 필요할 때 불러오며 오프라인 캐시 대상입니다. 계산할 수 없는 환경에서는 토큰 계산 불가로 표시합니다.

8.8 World Info JSON 내보내기

개별 로어북의 관리 메뉴에서 World Info(.json)로 내보내기를 선택할 수 있습니다. 프로젝트 관리 메뉴에서는 프로젝트 안의 여러 로어북을 선택해 하나의 JSON으로 묶을 수 있습니다.

여러 항목 내보내기 과정은 다음과 같습니다.

  1. 포함할 로어북을 체크합니다.
  2. 선택된 항목 수와 합산 토큰을 확인합니다.
  3. 파일 이름을 지정합니다.
  4. 하나의 entries 객체를 가진 SillyTavern World Info JSON을 저장합니다.

각 항목에는 제목, 키워드, 본문, constant, Depth, 기본 확률과 순서값 등이 들어갑니다.

8.9 World Info JSON 가져오기

사이드바의 열기에서 World Info JSON을 선택하면 entries 객체 또는 항목 배열을 읽습니다. 대상 프로젝트를 고르면 각 World Info 항목이 별도의 로어북 메모로 생성됩니다.

comment
→ 로어북 제목
key / keys
→ 발동 키워드
content
→ 본문
constant
→ 항상 활성
position: 4
→ 깊이 사용
depth
→ 깊이 값

8.10 로어북 합치기

같은 프로젝트에서 로어북을 두 개 이상 선택한 뒤 합치기를 누르면 원본을 삭제하지 않고 합친 로어북이라는 새 메모를 만듭니다. 본문은 빈 줄 두 개로 이어 붙입니다. 키워드, constant, Depth는 자동 병합하지 않으므로 새 메모에서 다시 설정해야 합니다.

9.1 기본 개념

로그 저장은 원문과 꾸며진 보기를 분리하는 메모 유형입니다.

편집 모드

보기 모드
선택한 템플릿의 정규식과 스타일을 적용한 결과만 보여줍니다.
  • 원문에는 꾸밈용 HTML을 직접 삽입하지 않습니다.
  • 템플릿을 바꾸거나 이름 가림을 적용해도 원문은 유지됩니다.

눈·연필 버튼으로 두 모드를 전환합니다.

9.2 정규식 디자인

템플릿은 일반적으로 다음 표식을 찾아 서로 다른 스타일을 적용합니다.

"큰따옴표 안쪽" '작은따옴표 안쪽' [대괄호 안쪽] **굵게 형식** *기울임 형식*

실제 인식 표식, 구분자 제거 여부, 색상과 테두리는 템플릿마다 다릅니다. 규칙은 템플릿 JSON의 배열 순서대로 적용됩니다.

9.3 이름 가림

로그에 등장하는 페르소나명, 본명, 애칭, 호칭 등을 보기 결과에서 다른 표시로 치환할 수 있습니다.

  1. 가릴 원본 이름에 이름을 입력합니다.
  2. 추가 이름이 있으면 + 원본 이름을 누릅니다.
  3. 필요하면 대체 표시를 입력합니다.
  4. 이름 가림 적용을 누릅니다.

세부 동작은 다음과 같습니다.

  • 원본 이름은 최대 20개입니다.
  • 같은 이름은 데이터 저장 시 중복 제거됩니다.
  • 여러 이름이 겹치면 긴 이름부터 처리합니다.
  • 이름 비교는 대소문자를 구분하는 정확한 문자열 검색입니다.
  • 단어 경계를 따로 보지 않으므로 다른 단어 안에 포함된 문자열도 치환될 수 있습니다.
  • 대체 표시를 비워 두면 템플릿이 지정한 기본 가림 문자열을 사용합니다.
  • 모든 이름은 같은 대체 문자열과 같은 템플릿 가림 스타일을 사용합니다.
  • 가림 스타일은 솔리드 칩, 아웃라인, 언더라인, 소프트 틴트, 마커 하이라이트의 5종 중에서 고릅니다.
  • 이름 가림은 본문뿐 아니라 꾸며진 로그의 제목 영역에도 적용됩니다.
  • 원문과 메모의 실제 제목 데이터는 바뀌지 않고 보기·복사·내보내기 결과에서만 가립니다.

9.4 기본 제공 템플릿 100종

템플릿 선택 창의 기본 탭에는 100종이 들어 있습니다. 목록은 이름순으로 정렬할 수 있으며, 별도의 100종 소개 보기 페이지에서 본문 샘플과 이름 가림 5종을 함께 비교할 수 있습니다.

사이드바 열기는 Android 파일 앱의 종류 분류에 관계없이 모든 파일을 선택할 수 있습니다. 앱 안에서 HTML · JSON · 일반 텍스트를 판별하며, 이진 파일은 열지 않습니다.

전체 이름은 부록 A에 정리되어 있습니다.

9.5 템플릿 선택 창

템플릿 창은 세 탭으로 구성됩니다.

창 안에서 할 수 있는 작업은 다음과 같습니다.

  • 이름 오름차순·내림차순 정렬
  • 템플릿을 길게 눌러 즐겨찾기 추가·해제

9.6 사용자 템플릿

템플릿 선택 창의 JSON 템플릿 업로드로 사용자 템플릿을 추가합니다.

주요 제한은 다음과 같습니다.

  • UTF-8 JSON 파일
  • 파일 크기 200KB 이하
  • kind: lumink-log-template
  • schemaVersion: 1
  • ID는 영문·숫자·하이픈으로 이루어진 3~64자
  • 기본 템플릿과 ID 중복 불가
  • 사용자 템플릿 최대 20개
  • 정규식 규칙 최대 20개
  • 허용된 CSS 속성만 사용
  • 외부 URL, @import, expression, javascript: 등 위험한 스타일 차단
  • 지나치게 복잡하거나 위험한 정규식 거부

사용자 템플릿을 메모에 적용하면 해당 템플릿의 스냅샷이 메모 안에도 저장됩니다. 나중에 사용자 템플릿 목록에서 삭제해도 이미 적용된 메모는 보존된 스냅샷으로 계속 렌더링할 수 있습니다.

전체 제작 규격은 로그 템플릿 제작 가이드 문서를 참고합니다.

9.7 인라인 HTML 코드 복사

메모 관리 메뉴의 인라인 HTML 코드 복사는 현재 템플릿 결과를 하나의 HTML 조각으로 만듭니다.

  • 배경, 테두리, 글자색, 문단 스타일을 모두 style 속성에 인라인 처리
  • 외부 스타일시트 없이 게시글 HTML 영역에 붙여넣기 쉬운 형태
  • 현재 제목과 본문에 이름 가림 적용
  • 원본 로그 데이터나 편집 UI는 포함하지 않음

게시판이 인라인 스타일을 제거하거나 허용 HTML을 제한하면 일부 디자인은 달라질 수 있습니다.

9.8 꾸며진 HTML 파일 저장

꾸며진 HTML 파일로 저장은 브라우저에서 독립적으로 열 수 있는 완성 문서를 만듭니다. 화면에 보이는 인라인 디자인과 함께 루미잉크 재가져오기용 로그 데이터도 문서 안에 포함합니다.

이 파일을 사이드바의 열기로 가져오면 대상 프로젝트에 새 로그 메모가 만들어지고, 원문·템플릿·이름 가림 설정을 다시 편집할 수 있습니다.

9.9 로그 합치기

같은 프로젝트의 로그를 두 개 이상 선택해 합치면 원본은 그대로 두고 합친 로그를 새로 만듭니다.

  • 본문은 빈 줄 두 개로 연결합니다.
  • 템플릿과 이름 가림 설정은 선택된 첫 번째 로그의 설정을 사용합니다.
  • 새 로그의 제목은 합친 로그로 잠깁니다.
  • 결과는 보기 모드로 열립니다.

10.1 편집 모드와 보기 모드

페르소나는 한 인물을 집중 관리하는 단일 카드이고, 다인 페르소나는 여러 페르소나 페이지를 한 메모에 묶어 관리하는 카드입니다. 둘은 동일한 데이터 구조를 공유하므로, 단일 카드는 메뉴에서 다인 페르소나로 확장할 수 있고 페이지가 하나인 다인 카드는 다시 단일 페르소나로 정리할 수 있습니다. 기존 데이터의 국문·영문 설정, 이미지·갤러리·태그·크리에이터 노트는 전환 중 보존됩니다.

10.2 한국어·영어 이중 양식

한국어와 English 탭은 각각 다음 데이터를 따로 가집니다.

  • 이름
  • 태그
  • 상세 설명
  • 상세 설명 토큰 수

이미지와 갤러리는 두 언어 탭이 공유합니다. 한국어 이름이 비어 있으면 영어 이름이 메모 자동 제목에 사용됩니다.

10.3 포트레이트와 정사각 썸네일

포트레이트
3:4 비율, 1200×1600으로 저장
정사각 썸네일
1:1 비율, 1080×1080으로 저장

이미지를 선택하면 자르기 화면이 열립니다. 이미지를 드래그해 위치를 옮기고 확대 슬라이더로 배율을 조절한 뒤 적용합니다. 정사각 썸네일이 없으면 보기 화면과 목록에서 포트레이트를 대신 사용합니다.

10.4 태그

쉼표 또는 Enter로 여러 태그를 추가할 수 있습니다. 중복 태그는 다시 추가하지 않으며 각 태그의 × 버튼으로 삭제합니다.

10.5 상세 설명 문법

페르소나와 캐릭터의 상세 설명은 전체 마크다운이 아니라 카드용 경량 구조 문법을 사용합니다.

입력보기 결과

- 라벨: 값라벨 미니 칩
- 항목삼각형 글머리표

10.6 토큰 수

한국어와 영어 상세 설명의 토큰을 각각 계산합니다. 보기 모드에서는 현재 선택한 언어의 토큰을 표시합니다.

10.7 갤러리

갤러리의 + 버튼에서 여러 이미지를 한 번에 선택할 수 있습니다. 각 이미지는 긴 변 최대 1600px로 줄여 저장합니다.

  • 이미지를 약 0.3초 누른 뒤 다른 위치로 끌어 순서 변경
  • 삭제 후 약 6초 동안 되돌리기 가능
  • 보기 모드에서 이미지를 눌러 확대
  • 확대 화면의 이전·다음 버튼, 모바일 좌우 스와이프, 키보드 좌우 화살표 지원
  • 확대 화면에서 인접 이미지를 미리 불러와 전환 지연 감소

10.8 빠른 복사

보기 모드의 상세 설명을 길게 누르면 현재 언어의 원본 상세 설명을 클립보드에 복사합니다.

10.10 페르소나 모드 HTML 내보내기

관리 메뉴에서 HTML로 저장을 누르면 밝게 또는 어둡게를 선택합니다. 앱의 현재 컬러 테마 기조를 유지하면서 카드 전체의 밝기만 결정합니다.

내보낸 HTML에는 다음이 포함됩니다.

  • 메모 제목
  • 포트레이트와 정사각 썸네일
  • 한국어·영어 이름, 태그, 상세 설명
  • 갤러리
  • 재가져오기용 페르소나 데이터

사이드바의 열기로 이 HTML을 선택하면 편집 가능한 새 페르소나 메모로 복원됩니다.

10.11 크리에이터 노트

페르소나 모드에도 인물 공개 설정과 분리된 크리에이터 노트가 있습니다. 제작 의도, 말투의 미세한 결, 관계성 설계, OOC 방지 메모처럼 공개 카드에 싣지 않을 작업 기록을 리치 텍스트로 정리합니다. 보기 화면의 크리에이터 노트 팝업에서는 닫기·코드 복사 등 조작을 안전하게 사용할 수 있습니다.

10.12 다인 페르소나로 확장

더보기 메뉴에서 페르소나 모음으로 확장을 선택하면 현재 인물을 첫 페이지로 유지한 채 다인 모드로 전환합니다. 새 메모 생성기에서는 이 형식을 다인 페르소나로 표기합니다. 국문·영문 설정, 태그, 이미지, 갤러리, 크리에이터 노트, 표지 정보는 삭제되지 않습니다.

11.1 페르소나 모드와의 관계

캐릭터는 한 캐릭터를 관리하는 단일 카드이고, 다인 캐릭터는 여러 캐릭터 페이지를 한 메모에 묶어 관리하는 카드입니다. 페이지마다 국문·영문 설정, 이미지, 갤러리, 크리에이터 노트를 독립적으로 보관합니다. 새 메모 생성기에서는 다인 형식을 다인 캐릭터로 표기하며, 일부 관리 메뉴에서는 데이터 구조 이름인 “캐릭터 모음”으로 표시될 수 있습니다.

11.2 캐릭터 페이지

상단 캐릭터 네비게이션 옆 + 버튼으로 새 페이지를 추가합니다. 페이지마다 다음 데이터가 독립적입니다.

  • 포트레이트
  • 정사각 썸네일
  • 갤러리
  • 한국어 이름·태그·상세 설명
  • 영어 이름·태그·상세 설명
  • 크리에이터 메모

캐릭터 페이지가 두 개 이상이면 관리 메뉴에서 현재 페이지를 삭제할 수 있습니다. 최소 한 페이지는 항상 남아 있어야 합니다. 페이지 삭제 직후 약 6초 동안 되돌릴 수 있습니다.

11.3 페이지 이동

  • 상단 썸네일 네비게이션에서 인물 선택
  • 보기 모드 하단의 이전·다음 화살표
  • 모바일 보기 모드에서 좌우 스와이프

편집 중 페이지를 바꾸면 현재 페이지를 먼저 저장합니다.

11.4 국문·영문 설정과 이미지

각 페이지의 한국어·영어 필드, 태그, 토큰 계산, 상세 설명 문법은 페르소나와 같습니다. 포트레이트는 3:4, 정사각 썸네일은 1:1 자르기를 사용합니다.

갤러리에서는 여러 이미지를 한 번에 추가하고, 보기 모드에서 확대·이전·다음·스와이프를 사용할 수 있습니다. 현재 캐릭터 갤러리는 페르소나 갤러리와 달리 드래그 순서 변경 기능을 제공하지 않습니다.

11.5 크리에이터 메모

크리에이터 메모는 공개용 캐릭터 설명과 별도로 제작 메모, 연출 주의점, 관계성 설계 등을 기록하는 리치 텍스트 영역입니다.

지원 도구는 다음과 같습니다.

HTML 소스 보기·편집굵게기울임밑줄취소선7단계 글자 크기글자 색5색 형광펜정렬글머리표·번호 목록이미지 삽입하이퍼링크코드 블록서식 지우기실행 취소

보기 모드에서는 크리에이터 메모가 있을 때 CREATOR NOTE 영역이 나타나며 크게 보기를 눌러 팝업으로 열 수 있습니다. 팝업의 코드 블록에는 복사 버튼이 붙습니다.

11.6 대표 썸네일

프로젝트 목록의 캐릭터 메모 카드에 사용할 대표 이미지를 별도로 지정할 수 있습니다.

  • 자동 선택: 첫 캐릭터 페이지의 정사각 썸네일, 포트레이트, 첫 갤러리 이미지 순으로 사용
  • 기존 캐릭터 페이지의 이미지 중 선택
  • 캐릭터 페이지와 별개인 새 대표 이미지 직접 업로드
  • 직접 업로드 이미지는 1:1로 잘라 1080×1080으로 저장

대표 썸네일은 프로젝트 카드 안에서 이 캐릭터 메모를 식별하기 위한 표지이며, 개별 캐릭터 페이지 이미지와는 별도입니다.

11.7 자동 제목

수동으로 메모 이름을 바꾸지 않은 경우 첫 캐릭터의 이름을 사용합니다. 페이지가 여러 개면 첫 캐릭터 이름 외 N명으로 표시합니다.

11.8 캐릭터 HTML 내보내기

관리 메뉴의 HTML로 저장에서 다음을 선택합니다.

  • 밝게 또는 어둡게
  • 크리에이터 메모 포함 또는 제외

크리에이터 메모의 기본값은 미포함입니다. 공유용 카드에 비공개 제작 메모가 실수로 섞이는 것을 줄이기 위한 설정입니다.

내보낸 파일은 여러 캐릭터 페이지를 전환하는 네비게이션을 포함하며, 루미잉크 재가져오기용 데이터도 내장합니다. 크리에이터 메모를 제외했다면 재가져오기 데이터에서도 해당 메모가 비어 있는 상태로 저장됩니다.

11.9 전환과 데이터 보존

단일 카드 → 다인 카드 전환은 첫 페이지를 그대로 유지한 뒤 추가 페이지를 열 수 있게 하는 방식입니다. 반대로 다인 카드 → 단일 카드는 페이지가 정확히 하나일 때 가능합니다. 페르소나와 캐릭터는 더보기 메뉴에서 서로 타입 전환도 할 수 있습니다. 전환 과정에서 이미지를 다시 압축하거나 텍스트를 재작성하지 않으며, 구형 데이터에서 앱이 알지 못하는 보조 필드도 호환 보관 영역에 남깁니다.

11+

아이디어 보드

아이디어 보드는 한 장의 넓은 캔버스 위에 생각의 조각을 붙이는 러프 콜라주형 메모입니다. 메모지, 이미지·GIF, 음악, 최대 30MB 영상, 첨부파일, 기존 메모로 이동하는 인용 링크를 원하는 위치에 놓고 크기·회전·색상까지 개별 조절합니다.

메모지는 본문을 누르면 미니 리치 도구막대가 열립니다. 글자 크기, 왼쪽·가운데·오른쪽 정렬, 굵게, 기울기, 밑줄, 취소선, 다섯 가지 형광펜, 서식 지우개를 지원합니다. 도구막대는 선택 핸들을 가리지 않도록 메모지 아래에 표시됩니다.

보드에 조각 추가하기

  1. 상단 + 버튼을 누르면 비어 있는 상단 여유 공간에 새 조각을 붙입니다.
  2. 캔버스의 빈 공간을 길게 누르면 해당 위치를 기준으로 추가 메뉴가 열립니다.
  3. 메모지, 이미지·GIF, 음악, 영상, 첨부파일, 내 글 인용 중 원하는 종류를 고릅니다.

이미지·GIF·음악·영상·첨부·캔버스 배경은 앱의 파일 저장소에 별도 보관됩니다. 투명 PNG와 GIF는 바깥 사각형 그림자 없이 이미지 윤곽의 그림자만 적용되어 스티커처럼 표시됩니다.

이동 · 크기 · 회전 · 옵션

조각을 길게 누르면 이동 모드가 시작됩니다. 이동 중에는 옵션 팝업이 열리지 않으며, 포인터 캡처가 끊기거나 화면이 전환되는 상황에서도 이동 상태가 남지 않도록 정리됩니다. 더블클릭·더블탭 또는 우클릭으로 옵션을 열어 이동 모드, 리사이즈, 비율 고정, 회전, 그림자, 프레임, 삭제를 관리합니다.

  • 리사이즈: 모든 조각의 크기를 조절합니다. 이미지·영상은 기본적으로 비율을 유지합니다. 음악·링크·첨부 조각은 겉 크기를 키우면 아이콘·글자·버튼도 함께 비율에 맞춰 커집니다.
  • 회전: 회전 핸들로 각도를 바꾸며, 회전 버튼을 빠르게 두 번 탭하면 0°로 복원합니다.
  • 프레임: 이미지와 영상 조각은 옵션에서 프로젝트 썸네일과 같은 전체 프레임 라이브러리·색상 선택기를 엽니다. 프레임 종류·색상을 조각별로 독립 선택할 수 있으며, 프레임은 바깥선에 걸쳐 표시됩니다. 가로·세로 리사이즈에서는 모서리와 장식이 고정되고 중앙 변만 늘어나는 9-slice 방식으로 형태를 보존합니다.
  • 저장: 이동 위치, 크기, 회전, z 순서, 색상, 메모지 내용과 서식이 조각 ID 기준으로 저장되어 보드/아티팩트 전환과 재진입 뒤에도 유지됩니다.

메모지와 컬러 테마

메모지는 형광 마커·테이프 노트·영수증·폴라로이드·라벨 스티커·연구 노트·클립 메모·봉랍 편지·네온 글래스·말풍선 메모지·동글 말풍선 메모지 등 디자인을 고를 수 있습니다. 말풍선 메모지는 중앙 꼬리와 안쪽 테두리를 사용하며, 동글 타입은 더 둥글고 통통한 형태입니다.

메모지·내 글 링크·첨부파일·음악·배경 오버레이는 테마 컬러, 크림색, 먹색, 저장한 색, 직접 선택 색을 사용할 수 있습니다. 자유 색상 선택기에는 HEX 입력과 함께 RGB 값 입력도 제공됩니다. 색상은 조각별로 저장되어 앱 테마와 독립적으로 유지됩니다.

내 글 링크와 첨부파일은 조각 본문을 선택·이동용으로 남기고, 끝단의 화살표·다운로드 버튼을 눌렀을 때만 실제 이동·다운로드가 실행됩니다.

음악 플레이어는 옵션에서 제목 표시를 켜거나 끌 수 있으며, 제목을 숨겨도 레코드 장식과 MUSIC 레이블은 남습니다. 표시 제목은 원본 파일명을 바꾸지 않고 보드에서만 변경됩니다.

동영상 조각은 어두운 매트 대신 밝은 카드형 플레이어 쉘과 작은 재생·별 장식으로 표시됩니다. 실제 재생·음량·전체화면 컨트롤은 브라우저 기본 플레이어를 그대로 사용합니다.

보드 배경과 캔버스 이미지

보드 배경에서는 청사진 그리드, 스크랩북, 코르크, 칠판, 모눈, 별자리도, 리넨, 크라프트, 다크 그리드, 기밀 문서판, 벚꽃, 터미널, 벨벳 등의 템플릿과 20종 단색 빈 캔버스를 선택할 수 있습니다. 선택 화면은 실제 질감이 보이는 썸네일 라이브러리로 표시됩니다.

이미지 배경에서는 제공 그림을, 이미지 업로드에서는 직접 올린 이미지를 사용합니다. 둘은 템플릿과 배타적으로 동작하며 한 번에 하나만 활성화됩니다. 이미지 위에는 테마·크림·먹색·직접 선택 색으로 오버레이를 올리고 투명도 0–92%를 조절할 수 있습니다.

배경 이미지의 기본 썸네일은 실제 사진을 업로드하거나 선택하기 전에도 고래 일러스트(제공 이미지)와 현재 테마색 업로드 마크(업로드 이미지)로 고정됩니다. 배경 이미지는 최대 30MB이며, 전체 백업·프로젝트 내보내기·복제·가져오기에서 함께 보존됩니다.

보드 / 아티팩트 보기

상단의 보드와 아티팩트 버튼으로 보기 방식을 전환합니다. 아티팩트 보기에서는 조각을 목록으로 정리해 종류·파일 정보·미리보기를 확인하고, ◉ 버튼으로 해당 조각만 팝업에서 크게 볼 수 있습니다. 캔버스에서의 위치 이동도 목록에서 관리할 수 있습니다.

꾸며진 HTML 내보내기와 다시 가져오기

보드 더보기 메뉴의 꾸며진 HTML로 내보내기를 누르면 현재 캔버스 배경, 위치, 크기, 회전, 메모지 디자인·색상·서식, 이미지·GIF·음악·영상·첨부를 포함한 독립 HTML 파일을 만듭니다. 이 파일은 브라우저에서 그대로 열어 감상하거나 배포할 수 있습니다.

내보낸 HTML 안에는 루미잉크 전용 보드 데이터와 첨부 원본도 함께 포함됩니다. 사이드바의 열기에서 같은 파일을 선택하면 새 아이디어 보드로 다시 가져오며, 첨부는 새 파일 ID로 복사됩니다. 단, 내 글 인용 링크는 같은 앱 안에 원본 메모가 남아 있을 때에만 다시 열 수 있습니다.

메모지 조절 UI

메모지를 선택하면 옵션·이동·회전·리사이즈 핸들이 디자인 바깥에 나타납니다. 디자인 표면과 편집 오버레이를 분리했으므로, 메모지 디자인이나 배치 위치와 관계없이 핸들이 잘리지 않습니다. 캔버스 상단 또는 가장자리 근처에서는 조작부가 자동으로 안쪽으로 피합니다.

다중 선택 · 그룹 · 스냅

다중선택 모드에서는 조각을 터치할 때마다 선택/해제가 누적됩니다. 선택한 조각은 즉시 외곽선과 광륜으로 표시되며, 잠긴 조각과 그룹 조각도 선택 여부를 바로 확인할 수 있습니다. 여러 조각을 그룹으로 묶으면 이동·크기 조절을 함께 할 수 있고, 잠금된 조각은 그룹화와 정렬 대상에서 제외됩니다.

스냅 ON 상태에서는 단일 조각뿐 아니라 그룹의 전체 외곽 경계도 캔버스 중심선 및 다른 조각의 좌·중앙·우 / 상·중앙·하 기준선에 맞춰집니다. 이동과 크기 조절 중에는 정렬 가이드가 표시되며, 그룹 내부 조각은 자기 자신에게 스냅되지 않습니다.

잠금 · 편집 도구 · 직접 색상

메모지의 잠금 메뉴에는 요소만 잠금과 전체 보호 잠금이 있습니다. 요소만 잠금은 이동·크기·회전·삭제·디자인 변경을 막지만 본문 편집은 유지합니다. 잠긴 메모를 선택하거나 편집할 때에도 서식 도구막대와 조작 핸들은 다른 조각보다 위 레이어에 표시됩니다.

직접 색상 선택기의 저장 버튼 옆 스포이드는 지원 브라우저와 HTTPS 환경에서 화면의 색을 가져옵니다. 네이티브 스포이드가 없는 환경 또는 일부 PWA에서는 화면 캡처 방식으로 전환되거나 기본 색상 선택기가 열릴 수 있습니다. 색을 가져온 뒤에는 반드시 적용 또는 저장을 눌러 반영합니다.

보기 모드 현재 장면 PNG

보기 모드의 더보기 메뉴에서 현재 보이는 보드 영역을 PNG로 저장할 수 있습니다. 캡처는 화면에 렌더링된 보드 DOM을 직접 래스터화하므로 메모지 장식·반복 질감·색상 그라데이션까지 유지하려고 시도합니다. 선택 테두리·핸들·잠금 배지·서식 도구막대는 이미지에 포함되지 않습니다. 이 기능은 PNG 전용이며, 변환 실패 시 SVG로 바꾸어 저장하지 않습니다.

템플릿을 직접 추가하기

추가 템플릿은 idea-board-templates.js에 키·이름·설명을 등록하고 idea-board-custom-templates.css에 같은 키의 CSS를 추가하는 방식으로 확장합니다. 선택 라이브러리의 카드와 썸네일은 자동으로 따라옵니다. 패키지에 포함된 idea-board-template-registry-guide.md와 idea-board-design-guide.md는 제작 규격과 AI 의뢰용 프롬프트를 제공합니다.

다중 선택·이동·합치기

13.1 선택 모드 진입

  • 홈의 프로젝트 카드 관리 메뉴에서 선택
  • 프로젝트 상세 상단의 선택 버튼
  • 메모 관리 메뉴의 선택

선택 모드 하단 바에서 선택 개수, 전체 선택, 합치기, 이동, 삭제를 사용할 수 있습니다.

13.2 여러 메모 이동

같은 프로젝트에서 선택한 메모들을 다른 프로젝트로 한꺼번에 이동합니다. 새 프로젝트를 만든 뒤 그 위치를 바로 선택할 수도 있습니다.

13.3 합치기 조건

  • 두 개 이상 선택해야 합니다.
  • 자유 메모·로어북·로그는 같은 메모 유형끼리만 합칠 수 있습니다.
  • 통합 인물 카드는 페르소나 모드와 캐릭터 모음 모드를 섞어 선택할 수 있습니다. 결과는 항상 새 캐릭터 모음입니다.
  • 코드 작업실과 아이디어 보드는 원본 코드·캔버스 배치 보존을 위해 합칠 수 없습니다.
  • 합치기는 원본을 삭제하지 않고 새 메모를 만듭니다.

13.4 유형별 합치기 결과

자유 메모
HTML 본문 사이에 빈 줄을 넣어 연결하고 첨부파일도 새 ID로 복제합니다.
로어북
원문만 빈 줄 두 개로 연결합니다. 키워드와 활성·깊이 설정은 초기값입니다.
로그
원문을 연결하고 첫 번째 선택 로그의 템플릿·이름 가림 설정을 사용합니다.

13.5 일괄 삭제

프로젝트 일괄 삭제는 각 프로젝트의 소속 메모와 첨부파일도 함께 삭제합니다. 메모 일괄 삭제는 선택한 메모와 연결 첨부파일을 삭제합니다. 삭제 후 약 6초 동안 한 번의 되돌리기 작업을 제공합니다.

열기와 형식 판별

사이드바 하단의 열기는 .html, .htm, .json, .md 및 일반 Markdown 확장자 파일을 받습니다. 파일 내용에 따라 다음처럼 처리합니다.

파일처리 방식
World Info JSON항목별 로어북으로 가져오기
일반 JSON코드 블록 메모로 보기 또는 코드 작업실에서 원문 그대로 열기
Markdown코드 작업실에서 원문 편집 · 렌더링 미리보기 · .md로 다시 저장
루미잉크 프로젝트 HTML메모와 첨부파일을 포함한 새 프로젝트로 복원
루미잉크 페르소나 HTML편집 가능한 페르소나로 복원
루미잉크 캐릭터 HTML편집 가능한 통합 인물 카드(캐릭터 모음 모드)로 복원
루미잉크 로그 HTML원문과 설정이 있는 로그로 복원
일반 HTML원본 보관용 코드 작업실 또는 안전 정화된 자유 메모 중 선택

프로젝트 HTML은 대상 프로젝트를 묻지 않고 새 프로젝트로 추가합니다. 그 밖의 메모·World Info 파일은 저장할 프로젝트를 먼저 선택합니다. 일반 JSON은 파일 확장자·MIME이 비어 있어도 내용 형식으로 다시 판별합니다. 가져온 데이터의 이미지, 텍스트 길이, ID, 메모 유형 등은 허용 범위로 정규화됩니다.

전체 백업 HTML은 일반 열기가 아니라 설정 → 백업 복원에서 가져와야 합니다.

설치형 PWA와 브라우저가 파일 핸들러 기능을 지원하는 환경에서는 운영체제의 연결 프로그램 또는 다른 앱으로 열기에서 루미잉크로 HTML 파일을 전달할 수도 있습니다. 전달받은 파일은 사이드바의 열기와 같은 형식 판별 절차를 거칩니다.

코드 작업실 선택 기준: HTML·JSON·Markdown 원문, 속성, 공백, 외부 리소스 구조까지 보존해야 하면 코드 작업실을 고릅니다. 내용을 안전한 리치 문서로 읽고 수정하는 목적이면 HTML을 자유 메모 문서로 가져옵니다.

  • 웹폰트 설정 / 정렬 방식

  • 저장되지 않은 임시 초안

16.2 전체 백업

설정 → 전체 백업은 다음 내용을 하나의 lumink-backup-날짜-시각.html 파일로 저장합니다.

  • 모든 프로젝트 · 모든 메모
  • 프로젝트 장식과 메모 설정
  • 본문에 포함된 이미지
  • 자유 메모 첨부파일

백업 HTML을 브라우저에서 열면 프로젝트와 메모 목록을 사람이 읽을 수 있는 요약이 보이고, 실제 복원 데이터는 문서 안의 JSON으로 포함됩니다.

전체 백업에는 앱 표시 설정, 사용자 로그 템플릿 목록, 템플릿 즐겨찾기가 별도 항목으로 포함되지 않습니다. 다만 사용자 템플릿을 적용한 로그 메모는 템플릿 스냅샷을 메모 안에 보관하므로 그 메모의 디자인은 백업에 포함됩니다.

16.3 백업 복원 방식

설정 → 백업 복원에서 전체 백업 HTML을 선택하면 두 방식을 고릅니다.

현재 데이터에 병합

  • 현재 프로젝트와 메모를 유지합니다.
  • 백업과 현재 데이터의 ID가 같으면 백업 내용으로 덮어씁니다.
  • 백업에만 있는 항목을 추가합니다.
  • 기본 메모함 중복은 기존 기본 메모함에 맞춰 정리합니다.

백업 시점으로 완전히 교체

  • 현재 프로젝트, 메모, 첨부파일을 지웁니다.
  • 선택한 백업 내용만 남깁니다.
  • 설정과 사용자 템플릿처럼 localStorage에 있는 값은 이 교체 대상이 아닙니다.

두 방식 모두 실제 복원 직전에 현재 작업 상태를 자동 백업으로 한 번 더 저장합니다. 프로젝트·메모·첨부파일 쓰기는 하나의 IndexedDB 트랜잭션으로 처리해 일부만 교체되는 상황을 줄입니다.

16.4 자동 백업

메모 저장을 계기로 자동 백업을 예약합니다. 저장이 매우 잦을 때에는 최소 약 18초 간격으로 묶어 실행합니다. 자동 백업에는 프로젝트, 메모, 첨부파일이 함께 들어갑니다.

  1. 설정 → 자동 백업을 누르면 스냅샷 목록이 먼저 열립니다. 아직 스냅샷이 없어도 같은 목록 화면에서 빈 상태 안내를 볼 수 있습니다.
  2. 각 스냅샷에는 날짜·시간, 프로젝트 수, 메모 수, 첨부 수가 표시됩니다. 복원을 누르면 현재 데이터에 병합 또는 백업 시점으로 완전 교체 중 방식을 고릅니다.
  3. 목록 하단의 보관 설정을 누르면 보관 수 설정 창이 열립니다. 이 창의 목록으로로 스냅샷 목록으로 되돌아갈 수 있습니다.

보관 수는 최소 1개~최대 15개이며 기본값은 10개입니다. 저장할 때마다 최신 스냅샷을 남기고, 보관 수를 줄이면 가장 오래된 스냅샷부터 즉시 정리합니다.

자동 백업은 같은 브라우저 저장공간 안에 있으므로 브라우저 사이트 데이터 자체를 삭제하면 함께 사라집니다. 기기 고장이나 브라우저 초기화에 대비하려면 외부 파일로 저장하는 전체 백업이 필요합니다.

16.5 저장되지 않은 임시 초안

자유 메모, 코드 작업실, 로어북, 로그, 통합 인물 카드, 아이디어 보드 편집 중에는 localStorage에 별도의 임시 초안을 만듭니다.

  • 메모별로 분리 저장
  • 최대 보관 기간 14일
  • JSON 문자열 기준 약 70만 자를 넘는 초안은 저장하지 않음
  • 대용량 이미지 전체가 아닌 텍스트 중심 안전망
  • 마지막 정상 저장보다 새롭고 실제 내용이 다를 때만 복구 여부 질문
  • 사용자가 복구 또는 버리기 선택

이 기능은 비정상 종료나 갑작스러운 새로고침을 보완하지만 정식 백업을 대체하지 않습니다.

16.6 저장공간 확인

설정의 저장공간은 브라우저가 제공하는 전체 사이트 사용량과 할당량, 자유 메모 첨부파일 수와 용량, 자동 백업 개수를 표시합니다. 브라우저가 전체 사용량 API를 제공하지 않으면 첨부파일과 백업 정보만 표시합니다.

이미지가 많으면 메모 데이터뿐 아니라 자동 백업도 커지므로 실제 사용량이 빠르게 증가할 수 있습니다.

16.7 데이터 초기화

데이터 초기화는 두 번의 확인을 거친 뒤 프로젝트, 메모, 첨부파일, 자동 백업을 모두 삭제합니다. 이 작업은 되돌릴 수 없습니다.

현재 구현에서 테마·폰트 같은 표시 설정과 사용자 로그 템플릿 등 localStorage 항목은 작업 데이터 초기화 대상에 포함되지 않습니다. 완전한 사이트 초기화가 필요하면 전체 백업 후 브라우저의 해당 사이트 데이터 삭제 기능을 사용해야 합니다.

16.8 새 타입의 백업 범위

  • 코드 작업실: 원본 소스 문자열과 제목·표식이 전체 백업 및 프로젝트 패키지에 포함됩니다.
  • 인물 카드: 페르소나·캐릭터의 단일/다인 모드, 페이지, 이미지, 갤러리, 크리에이터 노트, 호환 보관 필드가 포함됩니다.
  • 아이디어 보드: 조각 위치·크기·회전·색상·메모지 내용·캔버스 설정과 연결된 파일 Blob이 모두 포함됩니다.

개인정보와 보안상 주의

18.1 로컬 우선

프로젝트와 메모 본문을 루미잉크 자체 서버로 전송하는 동기화 기능은 없습니다. 자료는 현재 브라우저 저장소에 남습니다.

다만 웹폰트 프리셋이나 사용자가 지정한 외부 폰트 CSS를 불러올 때에는 해당 폰트 제공 서버에 일반적인 웹 요청이 발생할 수 있습니다. 이 요청에 메모 본문을 담아 보내지는 않습니다.

18.2 가져오기 정화

외부 HTML을 자유 메모로 가져올 때에는 허용된 태그와 스타일만 남깁니다. 스크립트, iframe, 폼, 임베드, 위험한 URL과 스타일은 제거합니다. 외부 원격 미디어는 가져오지 않고 루미잉크가 생성하는 안전한 base64 이미지와 인라인 동영상 형식만 수용합니다.

로그 템플릿은 임의 HTML과 JavaScript를 허용하지 않고, 스타일 속성 허용 목록과 정규식 복잡도 검사를 사용합니다.

18.3 직접 HTML 편집 주의

자유 메모와 크리에이터 메모의 HTML 소스 편집은 사용자가 직접 작성한 신뢰할 수 있는 내용에 사용합니다. 인터넷에서 가져온 불명확한 코드를 그대로 붙여넣지 않습니다.

18.4 브라우저 데이터 삭제 주의

다음 작업은 데이터 손실로 이어질 수 있습니다.

  • 브라우저의 쿠키·사이트 데이터 전체 삭제
  • 루미잉크 사이트 저장공간 삭제
  • 시크릿 모드에서 작성 후 창 닫기
  • 다른 도메인이나 다른 브라우저에서 기존 데이터가 자동으로 보일 것이라 기대하기
  • 운영체제 또는 브라우저 프로필 초기화

중요한 자료는 전체 백업 파일을 브라우저 밖의 안전한 위치에 보관합니다.

권장 사용 순서

처음 사용하는 경우 다음 순서가 가장 이해하기 쉽습니다.

  1. 작품 또는 세션 이름으로 프로젝트를 만듭니다.
  2. 프로젝트 태그, 색상, 썸네일을 지정합니다.
  3. 세계관은 로어북, 플레이어·단일 인물은 페르소나 또는 캐릭터, 여러 NPC·집단은 다인 페르소나 또는 다인 캐릭터에 기록합니다.
  4. 자유 메모에는 플롯·긴 문서·참고를, 원본 HTML·JSON·Markdown은 코드 작업실에, SillyTavern 치환 규칙은 정규식 작업실에, 흩어진 이미지·음악·링크·러프 아이디어는 아이디어 보드에 보관합니다.
  5. 대화 결과는 로그 저장에 원문 그대로 넣고 템플릿과 이름 가림을 적용합니다.
  6. 프로젝트와 메모를 고정해 자주 쓰는 항목을 위에 둡니다.
  7. 작업 중에는 자동 저장을 사용하되 큰 수정 뒤에는 Ctrl/⌘+S로 즉시 저장합니다.
  8. 작업이 끝난 날에는 전체 백업 HTML을 외부 폴더나 클라우드 드라이브에 저장합니다.

새 메모 생성기

  • 상단 설명은 생성할 메모 타입을 골라주세요이며, 자유 메모 빠른 생성 카드는 즉시 자유 메모 생성 흐름으로 들어갑니다. 기존 블루 그라데이션·흰색 아이콘/텍스트 디자인을 유지하며, 카드 아래 그림자는 짧고 더 흐리게만 표시됩니다.
  • 유형 선택은 캐릭터 / 작업실 / 아뜰리에 3탭입니다. 캐릭터에는 페르소나·다인 페르소나·캐릭터·다인 캐릭터, 작업실에는 로어북·정규식 작업실·코드 작업실, 아뜰리에에는 자유 메모·로그 저장·아이디어 보드가 있습니다.
  • 페르소나의 보조 설명은 단일 페르소나 카드 · 국문/영문 · 이미지로 표시합니다.
  • 탭 안의 타입 카드는 동일한 전체 폭으로 정렬되며, 카드 표면과 카드 안 아이콘 칩은 기존의 테마 배경색을 유지합니다.
  • 탭 안의 타입 카드는 기존의 카드 표면과 아이콘 칩 색상을 유지합니다. 각 카드는 내용 길이와 무관하게 팝업 폭을 끝까지 채우며, 제목은 기본 잉크색으로 표시됩니다.

정규식 작업실

  • 새 메모와 퀵메뉴의 메모 바로 만들기에서 정규식 작업실을 만들 수 있습니다. 기존 SillyTavern Regex JSON은 사이드바 열기에서 구조를 인식해 바로 작업실로 가져올 수 있습니다.
  • 샘플 자동은 OUT을 먼저 작성하지 않아도, 유효한 IN 정규식의 캡처 구조를 기준으로 sample 초안을 만듭니다.
  • 상태 문구는 편집과 미리보기의 제목 줄을 밀어내지 않는 작은 보조 정보로 표시됩니다. 단일 정상 결과는 매치로, 여러 결과는 N개 매치로 표시합니다.
  • 보기 전환 탭 아래의 별도 어두운 그라데이션 바탕을 제거했습니다. PREVIEW는 확인 전용 샌드박스 캔버스로 취급하므로, 미리보기 안쪽을 터치한 뒤에도 앱의 뒤로가기·사이드바·오른쪽 퀵메뉴 제스처를 계속 사용할 수 있습니다.
  • 내보내기 옵션은 트림아웃 → 영향을 미침 → 최소 깊이/최대 깊이 → 그 외 옵션 → Ephemerality → Macros in Find Regex 순서로 정리됩니다. 그 외 옵션에는 비활성화됨·편집 실행, Ephemerality에는 형식 표시만·형식 프롬프트만이 들어갑니다.

같은 원본 이름은 다섯 이름 치환 세트 중 한 곳에만 저장할 수 있습니다. 중복이 발견되면 저장 버튼이 비활성화되고, 먼저 등록한 세트를 명확히 안내합니다. 로그 상단의 다섯 세트 버튼은 간결한 타일로 정리했으며, 퀵메뉴 내장 아이콘은 기존보다 80% 크기로 보정해 프로젝트 썸네일과 균형을 맞췄습니다.

코드 작업실의 파일 저장에서 현재 원문을 .html 또는 .json으로 선택해 내려받을 수 있습니다. JSON 원본으로 연 작업실은 .json을 기본으로 제안하고, JSON 저장 전에는 문법을 검사합니다. 기존 Ctrl/⌘+S와 상단 저장 버튼은 앱 내부 저장을 유지하며, 파일 저장은 Ctrl/⌘+Shift+S로도 열 수 있습니다.

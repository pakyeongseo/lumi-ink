# 개발 빌드 v66.29 · 정규식 작업실 옵션 구조·매뉴얼 갱신

> 사용자 배포 표기는 **v1.5**를 유지합니다.
> 개발 빌드: **v66.29**

## 정규식 작업실 UI

- EDIT / PREVIEW / SPLIT 보기 전환을 감싸던 어두운 세그먼트 바탕을 제거했습니다. 비활성 탭은 화면 위에 별도 검은 판을 만들지 않고, 필요한 경우 호버만 은은하게 반응합니다.
- IN 검증과 PREVIEW의 매치·오류·대기 상태 문구를 제목 줄의 보조 정보 크기로 한 번 더 줄였습니다. Android의 텍스트 자동 확대가 상태 문구만 키우지 않도록 텍스트 크기 조정도 고정했습니다.

## 내보내기 옵션 정리

옵션 서랍의 순서를 아래처럼 재구성했습니다. 기존 저장 키와 JSON 구조는 바꾸지 않았습니다.

1. **트림아웃** — 한 줄에 하나씩 입력
2. **플레이스먼트** — 사용자 입력 / AI 출력 / 슬래시 명령 / 월드 인포 / Reasoning
3. **최소 깊이 / 최대 깊이** — 각각 제목을 둔 두 입력칸
4. **다른 옵션 | Macros in Find Regex** — 실행·표시 체크와 매크로 치환 선택을 한 행에서 비교

## 문서

- 상세 매뉴얼의 기능 개요, 목차, 내보내기 비교표, 입력 한도, 권장 작업 순서에 정규식 작업실을 반영했습니다.
- 정규식 JSON 가져오기, IN/OUT/sample, EDIT·PREVIEW·SPLIT, 샘플 도구, 캡처 확인, SillyTavern 옵션, JSON 내보내기 설명을 새 전용 섹션으로 추가했습니다.

## 호환성

- 기존 정규식 메모, 전체 백업, 프로젝트 패키지, SillyTavern Regex JSON 가져오기/내보내기 형식은 그대로 유지됩니다.
- 서비스 워커 캐시를 갱신해 PWA에서도 새 화면과 매뉴얼을 우선 받습니다.

# 개발 빌드 v66.24 · 정규식 작업실 재디자인

- 정규식 작업실의 EDIT 화면을 **세로 프레스 라인** 구조로 재구성했습니다. `IN → OUT → sample`이 ①②③ 번호 단계로 이어지고, 왼쪽 잉크 스파인이 흐름을 한 줄로 연결해 작업 순서가 한눈에 들어옵니다.
- `scriptName`을 **명패형 카드**로 승격하고, `IN`의 검증 상태와 `sample`의 미리보기 상태를 **씰 펠릿(상태 배지)** 으로 표시합니다. 정상/실패/대기 상태가 색으로 즉시 구분됩니다.
- `export options`를 **접이식 서랍**(export options)으로 분리해, 핵심 편집 영역(IN/OUT)과 내보내기 설정을 시각적으로 구분했습니다. 글로벌 `/g`는 토글 스위치, 옵션 체크는 칩 형태로 바뀌었습니다.
- 미리보기 탭의 캡처 결과(`$1`, `$2`, `$3`)를 **활자 슬러그** 형태로 정리해 그룹 번호와 값을 또렷하게 읽을 수 있습니다.
- 모든 색은 현재 테마 토큰(`--accent` 외)에서 가져오므로 다크·라이트 및 모든 커스텀 팔레트에 자동으로 맞춰집니다.
- **명칭·라벨·입력 항목 ID는 일절 변경하지 않았고**, 동작 로직(검증·미리보기·JSON 내보내기·저장)도 그대로입니다. 비주얼만 교체했습니다.
- 코드 작업실 등 다른 화면의 디자인은 변경하지 않았습니다.

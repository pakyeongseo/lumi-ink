## 현재 기능 기준

### 새 메모
- 상단 **자유 메모** 빠른 생성 카드로 바로 자유 메모 생성 흐름에 들어갈 수 있습니다.
- 메모 유형은 **캐릭터 / 작업실 / 아뜰리에** 3탭으로 구분됩니다.
  - 캐릭터: 페르소나, 다인 페르소나, 캐릭터, 다인 캐릭터
  - 작업실: 로어북, HTML 작업실
  - 아뜰리에: 자유 메모, 로그 저장, 아이디어 보드
- 탭 안 타입 카드는 기존의 색감 있는 카드 표면과 아이콘 칩을 유지합니다. 카드 폭은 내용 길이와 무관하게 동일하게 채워집니다.
- 상단 빠른 생성 카드는 기존 블루 그라데이션과 흰색 아이콘·텍스트 구성을 유지합니다. 제목은 **자유 메모**이며, 아래 그림자는 짧고 더 흐리게 조정했습니다.
- 페르소나 설명은 **단일 페르소나 카드 · 국문/영문 · 이미지**로 표기합니다.
- 상단 안내 문구는 **생성할 메모 타입을 골라주세요**로 표시합니다.
- 타입 카드와 카드 안 아이콘 칩은 기존 배경색을 유지합니다.

### 자동 백업
- 설정의 **자동 백업**은 스냅샷 목록을 먼저 엽니다.
- 목록 하단 **보관 설정**에서만 보관 수를 변경합니다.
- 보관 수는 **1~15개**, 기본값은 **10개**입니다.
- 보관 수를 줄이면 오래된 스냅샷부터 즉시 정리됩니다.

### 아이디어 보드
- 다중선택 상태가 터치 직후 눈에 보이도록 표시됩니다.
- 잠긴 메모의 서식 도구와 조작 핸들이 다른 조각 위에 유지됩니다.
- 그룹 이동·크기 조절에도 스냅과 정렬 가이드가 적용됩니다.
- 보기 모드의 현재 장면은 화면 DOM을 직접 래스터화한 **PNG**로 저장합니다. 실패할 경우 SVG로 대체 저장하지 않습니다.
- 직접 색상 선택의 스포이드는 지원 브라우저에서 네이티브 도구를, 그 외 환경에서는 가능한 대체 경로를 사용합니다.

## v64.8 개발 반영 · 오른쪽 엣지 퀵 메뉴

- 화면 오른쪽에서 슬라이드해 펼치는 5슬롯 퀵 메뉴를 추가했습니다.

- 삭제된 프로젝트/메모 바로가기는 자동 정리되고, 이동한 메모 바로가기는 새 프로젝트 경로를 따라갑니다.

## v64.9 후속 개선 · 퀵 메뉴

- 설정 → **퀵 메뉴 설정**에서 사용 여부를 켜고 끌 수 있습니다. 꺼도 슬롯 구성과 사용자 지정 썸네일은 그대로 보관됩니다.
- 기존 기본형과, 설명 없이 아이콘만 보이는 **미니형** 중 표시 방식을 선택할 수 있습니다. 기본값은 기존 기본형입니다.
- 라이트 테마의 퀵 메뉴 제목을 테마 기조에 맞춘 짙은 잉크색으로 다듬고, 주변 그림자는 훨씬 짧고 옅게 조정했습니다.
- 펼친 메뉴 안의 편집 버튼을 홈 화면과 통일된 설정 기어 아이콘으로 교체했습니다.


## v65 개발 반영 · 사용자 지정 컬러 테마

- 설정 → **컬러 테마**에서 프리셋 외에 **직접 지정**을 선택할 수 있습니다.
- 주조색과 보조색을 직접 정하면 밝게/어둡게 각각에 맞춘 강조색·표면·글자 대비가 자동으로 만들어집니다.
- 적용 전에는 화면 전체에 실시간 미리보기가 나타나며, 취소 시 기존 테마로 복귀합니다.
- 직접 지정 팔레트와 선택 상태는 IndexedDB, 전체 백업, 자동 백업에 함께 보관됩니다.


## 개발 빌드 v65.3 · 라이트 직접 지정 색상

- 사용자 지정 색상을 밝게 모드의 구조 색상 전용으로 재설계했습니다.
- 기존 프리셋의 로고·아이콘·기본 썸네일·버튼/퀵 메뉴 그라데이션은 유지됩니다.
- 배경, 카드, 상단바, 글자, 경계선을 항목별 HEX로 직접 입력할 수 있으며 다크 모드는 기존 프리셋을 그대로 사용합니다.


## 개발 빌드 v65.3 · 퀵메뉴 코드 아이콘·안정화

- 각 퀵메뉴 슬롯에서 **SVG 코드** 또는 **CSS + HTML 마크업** 아이콘을 직접 등록할 수 있습니다.
- 사용자 코드는 해당 슬롯 아이콘 컨테이너로만 범위가 한정되고, 위험한 실행 요소·외부 URL은 저장 과정에서 제거됩니다.
- 화면 전환 때 동일한 슬롯 DOM을 다시 만들지 않도록 바꿔 패널 좌표·크기 미세 흔들림을 줄였습니다.
- 퀵메뉴 외부를 누르면 자동으로 접히며, 손잡이와 그림자를 더 작고 가볍게 다듬었습니다.

# 개발 빌드 v66.12 · JSON 원문 파일 저장

- HTML 작업실의 **파일 저장**에서 확장자를 **`.html` 또는 `.json`**으로 직접 선택합니다.
- JSON 원문으로 열었던 작업실은 `.json`을 기본값으로 제안하며, HTML 작업실은 `.html`을 기본값으로 제안합니다.
- `.json` 선택 시 JSON 문법을 검사합니다. 오류가 있으면 다운로드하지 않고 오류 위치를 안내합니다.
- 기존 상단 저장 및 `Ctrl/⌘+S`는 앱 내부 저장을 유지합니다. 파일 저장 창은 작업 막대 버튼, 더보기 메뉴, `Ctrl/⌘+Shift+S`에서 엽니다.
